## 1. 设计概述

### 1.1 设计目标
设计NESOM系统7个MVP模块间的业务逻辑集成方案，确保跨模块业务流程的完整性、一致性和可靠性，包括：
- 定义模块间业务交互的工作流和状态机
- 设计跨模块事务的数据一致性保障机制
- 实现业务异常的处理和恢复策略
- 建立业务规则引擎支持动态业务逻辑
- 提供业务监控和审计追踪能力

### 1.2 设计原则
1. **业务完整性**：跨模块业务流程端到端完整，无断点
2. **最终一致性**：业务数据最终一致，容忍短暂不一致
3. **可追溯性**：业务操作全程可追溯，支持审计和排查
4. **容错性**：业务异常有明确处理路径和恢复机制
5. **可配置性**：业务规则可配置，支持动态调整
6. **性能可扩展**：支持业务量增长，水平扩展

### 1.3 核心业务集成场景
| 场景编号 | 业务场景 | 涉及模块 | 关键挑战 |
|----------|----------|----------|----------|
| BS-001 | 设备故障告警处理 | 设备监控 → 报警管理 → 巡检管理 | 实时性、优先级处理 |
| BS-002 | 巡检任务执行和结果回填 | 巡检管理 → 设备监控 → 报表统计 | 数据一致性、状态同步 |
| BS-003 | 用户权限变更同步 | 用户权限 → 所有模块 | 实时生效、权限回收 |
| BS-004 | 报表数据聚合计算 | 数据采集 → 报表统计 → 系统配置 | 大数据量、计算精度 |
| BS-005 | 系统配置动态生效 | 系统配置 → 所有模块 | 配置一致性、灰度发布 |

## 2. 跨模块工作流设计

### 2.1 工作流: 设备故障告警处理 (WF-ALERT-001)

#### 2.1.1 工作流概览
**触发条件**: 设备监控模块检测到设备故障或指标超阈值  
**涉及模块**: 设备监控 → 报警管理 → 巡检管理 (可选) → 用户通知  
**业务目标**: 及时识别故障、通知相关人员、生成处理任务  
**SLA要求**: 端到端处理时间 < 30秒，通知到达率 > 99%

#### 2.1.2 工作流状态机
```
[设备异常检测] → [告警规则匹配] → [告警级别判定] → [通知策略选择]
       ↓                ↓                ↓                ↓
[生成告警记录] → [实时通知发送] → [生成巡检任务] → [等待处理]
       ↓                ↓                ↓                ↓
[告警状态监控] → [处理结果回填] → [告警关闭] → [生成处理报告]
```

#### 2.1.3 详细步骤设计

**STEP 1: 设备异常检测**
- **触发**: 设备监控模块实时数据流分析
- **输入**: 设备指标数据流、阈值配置、历史基线
- **处理逻辑**:
  1. 实时计算设备指标与阈值对比
  2. 应用异常检测算法（3-sigma、趋势分析）
  3. 判断是否满足告警条件
- **输出**: 异常事件 `{device_id, metric_type, current_value, threshold, severity}`

**STEP 2: 告警规则匹配**
- **触发**: 异常事件进入报警管理模块
- **输入**: 异常事件、告警规则库、设备上下文
- **处理逻辑**:
  1. 匹配设备对应的告警规则（设备级 → 类型级 → 全局级）
  2. 应用抑制规则（同一设备5分钟内不重复告警）
  3. 应用升级规则（历史告警数量触发级别提升）
- **输出**: 告警规则匹配结果 `{rule_id, alert_level, notification_channels}`

**STEP 3: 告警级别判定**
- **触发**: 规则匹配完成
- **输入**: 规则匹配结果、业务时段配置、值班人员
- **处理逻辑**:
  1. 根据业务时段调整告警级别（非工作时间提升级别）
  2. 检查值班人员可用性（自动分配或轮询）
  3. 判定最终告警级别和负责人
- **输出**: 最终告警 `{alert_id, level, assigned_person, escalation_path}`

**STEP 4: 实时通知发送**
- **触发**: 告警生成完成
- **输入**: 告警详情、通知配置、接收人信息
- **处理逻辑**:
  1. 根据级别选择通知渠道（SMS/电话/APP推送/邮件）
  2. 消息模板渲染（包含设备详情、位置、建议措施）
  3. 并行发送到多个渠道，至少一个渠道成功
- **输出**: 通知发送结果 `{notification_id, channels_sent, channels_failed}`

**STEP 5: 生成巡检任务（可选）**
- **触发**: 告警级别为严重或需要现场处理
- **输入**: 告警详情、设备信息、可用巡检人员
- **处理逻辑**:
  1. 根据设备类型生成标准巡检项
  2. 自动分配巡检人员（基于位置、技能、当前负载）
  3. 设置任务优先级和截止时间
- **输出**: 巡检任务 `{task_id, assignee, priority, deadline, checklist}`

**STEP 6: 告警状态监控**
- **触发**: 告警进入活动状态
- **输入**: 告警状态、处理进度、超时配置
- **处理逻辑**:
  1. 监控告警确认时间（严重告警15分钟未确认升级）
  2. 监控处理进度（定期检查处理状态）
  3. 自动升级机制（超时未处理自动升级）
- **输出**: 状态变更事件 `{alert_id, old_status, new_status, reason}`

**STEP 7: 处理结果回填**
- **触发**: 巡检任务完成或手动关闭告警
- **输入**: 处理结果、解决措施、根本原因
- **处理逻辑**:
  1. 验证处理结果完整性（必须填写解决措施）
  2. 更新设备状态（如设备恢复正常）
  3. 记录根本原因分析（用于预防改进）
- **输出**: 告警关闭事件 `{alert_id, resolution, root_cause, closed_by}`

**STEP 8: 生成处理报告**
- **触发**: 告警关闭后
- **输入**: 告警全生命周期数据、处理记录
- **处理逻辑**:
  1. 聚合告警处理时间线
  2. 计算MTTR（平均修复时间）
  3. 生成根本原因分析报告
- **输出**: 告警分析报告 `{report_id, alert_summary, performance_metrics, recommendations}`

#### 2.1.4 异常处理分支

**分支A: 通知发送失败**
- **条件**: 所有通知渠道发送失败
- **处理逻辑**:
  1. 重试3次，每次间隔1分钟
  2. 升级到备用通知渠道（企业微信 → 电话）
  3. 记录失败日志，人工介入
- **恢复路径**: 人工确认后重新发送或标记为已处理

**分支B: 巡检任务分配失败**
- **条件**: 无可用巡检人员或分配冲突
- **处理逻辑**:
  1. 扩大分配范围（全公司范围）
  2. 降低技能要求匹配
  3. 创建待分配任务队列
- **恢复路径**: 管理人员手动分配

**分支C: 处理超时**
- **条件**: 告警超过处理时限未解决
- **处理逻辑**:
  1. 自动升级告警级别
  2. 通知上级管理人员
  3. 生成超时告警事件
- **恢复路径**: 管理人员介入处理

### 2.2 工作流: 巡检任务执行和结果回填 (WF-INSPECTION-001)

#### 2.2.1 工作流概览
**触发条件**: 定时任务生成或手动创建巡检任务  
**涉及模块**: 巡检管理 → 设备监控 → 报表统计 → 用户权限  
**业务目标**: 规范巡检过程、准确记录结果、及时发现问题  
**SLA要求**: 任务执行率 > 95%，结果回填及时率 > 90%

#### 2.2.2 工作流状态机
```
[任务生成] → [任务分配] → [任务接收] → [现场执行]
    ↓            ↓            ↓            ↓
[数据采集] → [结果记录] → [异常上报] → [审核确认]
    ↓            ↓            ↓            ↓
[状态同步] → [报告生成] → [知识沉淀] → [任务关闭]
```

```python
def assign_inspection_task(task, available_personnel):
    # 权重计算：技能匹配(40%) + 位置距离(30%) + 当前负载(20%) + 历史完成率(10%)
    scores = []
    for person in available_personnel:
        skill_score = calculate_skill_match(task.skills_required, person.skills)
        distance_score = calculate_distance(task.location, person.current_location)
        load_score = calculate_load_factor(person.current_tasks)
        history_score = person.completion_rate
        
        total_score = (
            skill_score * 0.4 +
            distance_score * 0.3 + 
            load_score * 0.2 +
            history_score * 0.1
        )
        scores.append((person, total_score))
    
    # 选择最高分且接受阈值以上的人员
    best_person = max(scores, key=lambda x: x[1])
    if best_person[1] >= ASSIGNMENT_THRESHOLD:
        return best_person[0]
    else:
        return None  # 需要人工分配
```

```python
def validate_inspection_result(device, checklist_results):
    validation_rules = {
        'voltage': {'min': 210, 'max': 250, 'unit': 'V'},
        'temperature': {'max': 85, 'unit': '℃'},
        'noise_level': {'max': 75, 'unit': 'dB'}
    }
    
    violations = []
    for item in checklist_results:
        if item.metric in validation_rules:
            rule = validation_rules[item.metric]
            if 'min' in rule and item.value < rule['min']:
                violations.append(f"{item.metric}低于最小值{rule['min']}{rule['unit']}")
            if 'max' in rule and item.value > rule['max']:
                violations.append(f"{item.metric}高于最大值{rule['max']}{rule['unit']}")
    
    return {
        'is_valid': len(violations) == 0,
        'violations': violations,
        'severity': 'warning' if len(violations) <= 2 else 'error'
    }
```

#### 4.2.2 异常处理
```python
class DeviceCreationSaga:
    def execute(self, device_data):
        steps = [
            self.create_device_record,
            self.init_data_collection,
            self.set_default_thresholds,
            self.generate_qrcode,
            self.send_notification
        ]
        
        compensations = []  # 记录已完成的步骤用于补偿
        
        for i, step in enumerate(steps):
            try:
                result = step(device_data)
                compensations.append(i)  # 记录成功步骤
            except Exception as e:
                # 执行补偿操作
                self.compensate(compensations, device_data)
                raise SagaExecutionError(f"步骤{i+1}失败: {str(e)}")
        
        return {"status": "success", "device_id": device_data.id}
    
    def compensate(self, completed_steps, device_data):
        # 逆序执行补偿操作
        for step_index in reversed(completed_steps):
            compensation_func = self.get_compensation(step_index)
            try:
                compensation_func(device_data)
            except Exception as e:
                # 补偿失败记录日志，继续执行其他补偿
                log_error(f"步骤{step_index+1}补偿失败: {e}")
```

### 4.3 数据一致性保障

#### 4.3.1 版本控制机制
```python
class VersionedEntity:
    def __init__(self):
        self.version = 1
        self.last_updated = None
    
    def update(self, new_data, expected_version=None):
        # 乐观锁检查
        if expected_version and self.version != expected_version:
            raise ConcurrentModificationError(
                f"版本冲突: 期望版本{expected_version}, 当前版本{self.version}"
            )
        
        # 应用更新
        self.apply_changes(new_data)
        self.version += 1
        self.last_updated = datetime.now()
        
        # 发布变更事件
        self.publish_change_event()
        
        return self.version
```

#### 4.3.2 最终一致性检查
```python
def check_data_consistency(source_module, target_module, entity_type):
    # 1. 查询源模块数据
    source_data = source_module.query_entities(entity_type)
    
    # 2. 查询目标模块数据  
    target_data = target_module.query_entities(entity_type)
    
    # 3. 对比关键字段
    inconsistencies = []
    for src in source_data:
        tgt = find_matching_target(src, target_data)
        if not tgt:
            inconsistencies.append({
                'type': 'missing',
                'entity_id': src.id,
                'details': '目标模块缺少此实体'
            })
        elif not compare_entities(src, tgt):
            inconsistencies.append({
                'type': 'mismatch',
                'entity_id': src.id,
                'details': f"字段不匹配: {get_field_differences(src, tgt)}"
            })
    
    # 4. 生成一致性报告
    report = {
        'check_time': datetime.now(),
        'source_module': source_module.name,
        'target_module': target_module.name,
        'entity_type': entity_type,
        'total_checked': len(source_data),
        'inconsistencies': inconsistencies,
        'consistency_rate': 1 - len(inconsistencies) / len(source_data)
    }
    
    # 5. 触发修复（如果配置了自动修复）
    if inconsistencies and AUTO_REPAIR_ENABLED:
        repair_inconsistencies(inconsistencies)
    
    return report
```

### 6.2 重试和补偿机制
```python
class ResilientBusinessOperation:
    def __init__(self, max_retries=3, backoff_factor=2):
        self.max_retries = max_retries
        self.backoff_factor = backoff_factor
    
    def execute_with_retry(self, operation, compensation=None):
        last_exception = None
        
        for attempt in range(self.max_retries):
            try:
                return operation()
            except TransientError as e:  # 可重试异常
                last_exception = e
                if attempt < self.max_retries - 1:
                    wait_time = self.backoff_factor ** attempt
                    time.sleep(wait_time)
                    continue
            except PermanentError as e:  # 不可重试异常
                if compensation:
                    compensation()
                raise e
        
        # 所有重试都失败
        if compensation:
            compensation()
        raise OperationFailedError(
            f"操作重试{self.max_retries}次均失败",
            cause=last_exception
        )
```

### 9.2 批量处理优化
```python
class BatchProcessor:
    def __init__(self, batch_size=100, max_concurrent=10):
        self.batch_size = batch_size
        self.max_concurrent = max_concurrent
        self.semaphore = Semaphore(max_concurrent)
    
    async def process_batch(self, items, process_func):
        results = []
        errors = []
        
        # 分批处理
        for i in range(0, len(items), self.batch_size):
            batch = items[i:i + self.batch_size]
            
            async with self.semaphore:
                try:
                    batch_result = await process_func(batch)
                    results.extend(batch_result)
                except Exception as e:
                    errors.append({
                        'batch_index': i // self.batch_size,
                        'error': str(e),
                        'items': batch
                    })
        
        return {
            'total_processed': len(results),
            'successful': results,
            'failed': errors,
            'success_rate': len(results) / len(items) if items else 1.0
        }
```
