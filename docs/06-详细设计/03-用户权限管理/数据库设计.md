**技术栈**: Vue 3.4 + Python 3.11 + FastAPI + MySQL 8.0 + Redis + Docker

## 1. 设计概述

### 1.1 设计目标
用户权限管理模块是NESOM系统的安全核心，负责：
- 用户身份认证和会话管理
- 基于角色的访问控制（RBAC）
- 组织架构和部门管理
- 操作审计和日志记录
- 多因素认证和安全策略
- 密码策略和账户安全

### 1.2 设计原则
1. **安全性第一**：密码加密存储、防暴力破解、会话安全
2. **性能优化**：高频认证操作优化，索引设计合理
3. **扩展性**：支持多级组织架构、灵活权限分配
4. **合规性**：满足等保2.0三级要求，日志保留180天
5. **易用性**：管理员易于管理，用户易于使用

### 1.3 与其他模块关系
- **基础服务**：为所有模块提供统一的认证和授权服务
- **数据提供**：用户信息、角色权限数据供各模块使用
- **审计集成**：集中记录所有模块的操作审计日志
- **通知集成**：账户安全事件通知（登录异常、密码修改等）

## 2. 表结构详细设计

### 2.1 用户表 (users)
系统用户主表，存储用户基础信息和认证凭据。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID() | 主键 |
| username | VARCHAR(50) | UNIQUE NOT NULL | - | 用户名，唯一标识 |
| email | VARCHAR(100) | UNIQUE NOT NULL | - | 邮箱，用于登录和通知 |
| phone | VARCHAR(20) | UNIQUE | NULL | 手机号，支持短信验证 |
| password_hash | VARCHAR(255) | NOT NULL | - | 密码哈希（bcrypt） |
| real_name | VARCHAR(50) | NOT NULL | - | 真实姓名 |
| avatar | VARCHAR(255) | - | NULL | 头像URL |
| department_id | VARCHAR(36) | FOREIGN KEY | NULL | 所属部门ID |
| position | VARCHAR(50) | - | NULL | 职位 |
| employee_id | VARCHAR(50) | UNIQUE | NULL | 员工工号 |
| status | ENUM('active','inactive','locked','pending') | NOT NULL INDEX | 'active' | 账户状态 |
| is_superadmin | BOOLEAN | NOT NULL | FALSE | 是否超级管理员 |
| last_login_at | DATETIME | - | NULL | 最后登录时间 |
| last_login_ip | VARCHAR(45) | - | NULL | 最后登录IP |
| last_password_change | DATETIME | - | NULL | 最后密码修改时间 |
| password_expires_at | DATETIME | - | NULL | 密码过期时间 |
| failed_login_attempts | INT | NOT NULL | 0 | 连续失败登录次数 |
| lockout_until | DATETIME | - | NULL | 锁定截止时间 |
| mfa_enabled | BOOLEAN | NOT NULL | FALSE | 是否启用MFA |
| mfa_secret | VARCHAR(255) | - | NULL | MFA密钥（加密存储） |
| mfa_backup_codes | JSON | - | NULL | MFA备份代码（加密） |
| timezone | VARCHAR(50) | - | 'Asia/Shanghai' | 时区 |
| locale | VARCHAR(10) | - | 'zh-CN' | 语言区域 |
| theme | VARCHAR(20) | - | 'light' | 主题偏好 |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| deleted_at | DATETIME | - | NULL | 软删除时间 |

**外键约束**：
- `FOREIGN KEY (department_id) REFERENCES departments(id) ON DELETE SET NULL`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 唯一索引：`UNIQUE KEY uk_username (username)`, `UNIQUE KEY uk_email (email)`, `UNIQUE KEY uk_phone (phone)`, `UNIQUE KEY uk_employee_id (employee_id)`
- 查询索引：`INDEX idx_status (status)`, `INDEX idx_department (department_id)`, `INDEX idx_created_at (created_at)`
- 复合索引：`INDEX idx_username_status (username, status)` 用于登录查询

**安全设计**：
1. password_hash使用bcrypt算法，cost factor=12
2. mfa_secret使用AES-256-GCM加密存储
3. mfa_backup_codes使用HMAC-SHA256哈希存储
4. 敏感字段（email, phone）在日志中脱敏

**业务规则**：
1. username只能包含字母、数字、下划线，长度3-50
2. email必须验证格式，支持后续二次验证
3. phone格式验证（中国手机号+国际）
4. 账户状态流转：pending → active → (inactive/locked) → active
5. 密码策略：最小长度8，必须包含大小写字母和数字，90天强制修改
6. 连续5次登录失败锁定账户30分钟
7. 超级管理员账户不可删除，权限不受限

### 2.2 角色表 (roles)
系统角色定义表，支持RBAC模型。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID() | 主键 |
| role_code | VARCHAR(50) | UNIQUE NOT NULL | - | 角色编码，程序识别 |
| role_name | VARCHAR(100) | NOT NULL | - | 角色名称 |
| description | TEXT | - | NULL | 角色描述 |
| role_type | ENUM('system','custom','department') | NOT NULL | 'custom' | 角色类型 |
| is_default | BOOLEAN | NOT NULL | FALSE | 是否默认角色（新用户自动分配） |
| is_protected | BOOLEAN | NOT NULL | FALSE | 是否受保护（不可删除） |
| max_users | INT | - | NULL | 最大用户数限制 |
| priority | INT | NOT NULL | 0 | 角色优先级（权限冲突时取高） |
| data_scope | ENUM('all','department','self','custom') | NOT NULL | 'self' | 数据权限范围 |
| scope_expression | JSON | - | NULL | 数据权限表达式 |
| created_by | VARCHAR(36) | FOREIGN KEY | NULL | 创建人 |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**外键约束**：
- `FOREIGN KEY (created_by) REFERENCES users(id) ON DELETE SET NULL`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 唯一索引：`UNIQUE KEY uk_role_code (role_code)`
- 查询索引：`INDEX idx_role_type (role_type)`, `INDEX idx_is_default (is_default)`
- 排序索引：`INDEX idx_priority (priority DESC)`

**业务规则**：
1. 角色编码格式：`ROLE_{模块}_{功能}`，如`ROLE_DEVICE_VIEW`
2. 系统角色（system）内置不可修改：`ROLE_SUPERADMIN`, `ROLE_ADMIN`, `ROLE_USER`
3. 部门角色（department）自动继承部门权限
4. 数据权限范围：
   - all：所有数据
   - department：本部门及下级部门数据
   - self：仅本人创建的数据
   - custom：自定义表达式
5. 角色删除时自动解除用户关联

### 2.3 权限表 (permissions)
系统权限定义表，支持操作级和数据级权限。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID() | 主键 |
| perm_code | VARCHAR(100) | UNIQUE NOT NULL | - | 权限编码，全局唯一 |
| perm_name | VARCHAR(100) | NOT NULL | - | 权限名称 |
| module | VARCHAR(50) | NOT NULL INDEX | - | 所属模块 |
| resource | VARCHAR(50) | NOT NULL INDEX | - | 资源类型 |
| action | ENUM('create','read','update','delete','execute','manage') | NOT NULL | 'read' | 操作类型 |
| description | TEXT | - | NULL | 权限描述 |
| is_system | BOOLEAN | NOT NULL | FALSE | 是否系统权限 |
| depends_on | JSON | - | NULL | 依赖权限列表 |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 唯一索引：`UNIQUE KEY uk_perm_code (perm_code)`
- 查询索引：`INDEX idx_module_resource (module, resource)`, `INDEX idx_module (module)`, `INDEX idx_resource (resource)`
- 复合索引：`INDEX idx_module_resource_action (module, resource, action)`

**业务规则**：
1. 权限编码格式：`{模块}:{资源}:{操作}`，如`device:monitor:read`
2. 模块对应系统功能模块：`auth`, `device`, `workorder`, `report`等
3. 资源对应数据实体：`user`, `role`, `device`, `alert`等
4. 操作类型：
   - create：创建资源
   - read：查看资源
   - update：修改资源
   - delete：删除资源
   - execute：执行操作
   - manage：管理权限
5. 依赖权限：如`device:monitor:update`依赖`device:monitor:read`

### 2.4 用户角色关联表 (user_roles)
用户和角色的多对多关联表。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | BIGINT | PRIMARY KEY AUTO_INCREMENT | - | 自增主键 |
| user_id | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 用户ID |
| role_id | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 角色ID |
| assigned_by | VARCHAR(36) | FOREIGN KEY | NULL | 分配人 |
| assigned_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 分配时间 |
| expires_at | DATETIME | - | NULL | 过期时间 |
| is_active | BOOLEAN | NOT NULL | TRUE | 是否生效 |
| notes | TEXT | - | NULL | 分配说明 |

**外键约束**：
- `FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE`
- `FOREIGN KEY (role_id) REFERENCES roles(id) ON DELETE CASCADE`
- `FOREIGN KEY (assigned_by) REFERENCES users(id) ON DELETE SET NULL`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 唯一索引：`UNIQUE KEY uk_user_role (user_id, role_id)` 防止重复分配
- 查询索引：`INDEX idx_user_id (user_id)`, `INDEX idx_role_id (role_id)`
- 过期索引：`INDEX idx_expires_at (expires_at)` 用于定时清理

**业务规则**：
1. 一个用户可以有多个角色，角色权限取并集
2. 角色过期后自动失效，需重新分配
3. 超级管理员角色不受权限限制
4. 角色分配记录审计日志

### 2.5 角色权限关联表 (role_permissions)
角色和权限的多对多关联表。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | BIGINT | PRIMARY KEY AUTO_INCREMENT | - | 自增主键 |
| role_id | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 角色ID |
| permission_id | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 权限ID |
| granted_by | VARCHAR(36) | FOREIGN KEY | NULL | 授权人 |
| granted_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 授权时间 |
| is_deny | BOOLEAN | NOT NULL | FALSE | 是否拒绝权限（黑名单） |
| condition | JSON | - | NULL | 权限条件表达式 |

**外键约束**：
- `FOREIGN KEY (role_id) REFERENCES roles(id) ON DELETE CASCADE`
- `FOREIGN KEY (permission_id) REFERENCES permissions(id) ON DELETE CASCADE`
- `FOREIGN KEY (granted_by) REFERENCES users(id) ON DELETE SET NULL`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 唯一索引：`UNIQUE KEY uk_role_permission (role_id, permission_id)`

**业务规则**：
1. 一个角色可以拥有多个权限
2. 支持黑名单机制（is_deny=true），优先级高于允许
3. 权限条件表达式支持基于时间、数据属性等动态判断
4. 系统角色权限不可修改

| parent_id | VARCHAR(36) | FOREIGN KEY | NULL | 父部门ID |

| manager_id | VARCHAR(36) | FOREIGN KEY | NULL | 部门负责人 |
| sort_order | INT | NOT NULL | 0 | 显示顺序 |

| email | VARCHAR(100) | - | NULL | 部门邮箱 |

| description | TEXT | - | NULL | 部门描述 |
| is_active | BOOLEAN | NOT NULL | TRUE | 是否有效 |

| level | INT | NOT NULL | 0 | 层级深度 |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |

- 主键索引：`PRIMARY KEY (id)`

- 路径索引：`INDEX idx_path (path(255))` 用于快速查询子树

**业务规则**：
1. 部门编码格式：根据层级自动生成，如`DEPT-001-001`
2. 部门类型：
   - company：公司
   - division：事业部
   - department：部门
   - team：小组
   - station：场站（物理位置）
3. 树形结构最大深度10层
4. path字段格式：`/根ID/父ID/当前ID`，便于查询子树
5. 部门停用时，下属用户不能分配新任务

### 2.7 用户会话表 (user_sessions)
用户登录会话管理表，支持多端登录。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID() | 主键 |
| user_id | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 用户ID |
| session_token | VARCHAR(255) | UNIQUE NOT NULL | - | 会话令牌（JWT ID） |
| refresh_token | VARCHAR(255) | UNIQUE | NULL | 刷新令牌 |
| client_type | ENUM('web','mobile','api','desktop') | NOT NULL | 'web' | 客户端类型 |
| client_info | JSON | - | NULL | 客户端信息（UA、IP等） |
| login_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 登录时间 |
| last_activity_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 最后活动时间 |
| expires_at | DATETIME | NOT NULL | - | 过期时间 |
| revoked_at | DATETIME | - | NULL | 撤销时间 |
| revocation_reason | VARCHAR(50) | - | NULL | 撤销原因 |
| ip_address | VARCHAR(45) | NOT NULL | - | 登录IP |
| user_agent | TEXT | - | NULL | 用户代理 |
| geo_location | JSON | - | NULL | 地理位置数据 |
| mfa_verified | BOOLEAN | NOT NULL | FALSE | MFA是否已验证 |

**外键约束**：
- `FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 唯一索引：`UNIQUE KEY uk_session_token (session_token)`, `UNIQUE KEY uk_refresh_token (refresh_token)`
- 查询索引：`INDEX idx_user_id (user_id)`, `INDEX idx_expires_at (expires_at)`, `INDEX idx_revoked_at (revoked_at)`
- 复合索引：`INDEX idx_user_active (user_id, revoked_at, expires_at)` 用于查找有效会话

**安全设计**：
1. session_token为JWT的jti claim，Redis中存储黑名单
2. refresh_token使用加密存储，可轮换
3. 客户端信息包含设备指纹，用于异常检测
4. 会话过期时间：access_token 15分钟，refresh_token 7天

**业务规则**：
1. 每个用户最多允许5个并发会话
2. 会话超过30分钟无活动自动延长（滑动过期）
3. 密码修改后所有会话自动撤销
4. 管理员可强制撤销用户会话

### 2.8 审计日志表 (audit_logs)
系统操作审计表，记录所有关键操作。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | BIGINT | PRIMARY KEY AUTO_INCREMENT | - | 自增主键 |
| event_time | DATETIME(3) | NOT NULL INDEX | CURRENT_TIMESTAMP(3) | 事件时间（毫秒精度） |
| user_id | VARCHAR(36) | FOREIGN KEY | NULL | 操作用户ID |
| username | VARCHAR(50) | NOT NULL INDEX | - | 用户名（冗余，避免关联查询） |
| event_type | VARCHAR(50) | NOT NULL INDEX | - | 事件类型 |
| event_subtype | VARCHAR(50) | - | NULL | 事件子类型 |
| resource_type | VARCHAR(50) | NOT NULL INDEX | - | 资源类型 |
| resource_id | VARCHAR(100) | NOT NULL | - | 资源ID |
| resource_name | VARCHAR(255) | - | NULL | 资源名称（冗余） |
| action | VARCHAR(20) | NOT NULL | - | 操作：create/read/update/delete/execute |
| old_values | JSON | - | NULL | 变更前值 |
| new_values | JSON | - | NULL | 变更后值 |
| changes | JSON | - | NULL | 变更摘要 |
| ip_address | VARCHAR(45) | - | NULL | 操作IP |
| user_agent | TEXT | - | NULL | 用户代理 |
| request_id | VARCHAR(100) | INDEX | NULL | 请求ID（跟踪链路） |
| status | ENUM('success','failed','denied') | NOT NULL | 'success' | 操作状态 |
| error_code | VARCHAR(50) | - | NULL | 错误码（失败时） |
| error_message | TEXT | - | NULL | 错误信息 |
|耗时_ms | INT | - | NULL | 操作耗时（毫秒） |
| tags | JSON | - | NULL | 标签（分类、优先级等） |

**外键约束**：
- `FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE SET NULL`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 时间索引：`INDEX idx_event_time (event_time DESC)` 主查询路径
- 用户索引：`INDEX idx_user_event (user_id, event_time DESC)`
- 资源索引：`INDEX idx_resource (resource_type, resource_id, event_time DESC)`
- 复合索引：`INDEX idx_type_time (event_type, event_time DESC)`
- 请求追踪：`INDEX idx_request_id (request_id)`

**分区策略**：
- 按月分区：`PARTITION BY RANGE (TO_DAYS(event_time))`
- 保留180天数据（等保2.0要求）
- 历史数据归档到审计历史表

**业务规则**：
1. 记录所有数据变更操作（CUD）
2. 记录所有权限相关操作（登录、授权、访问控制）
3. 记录所有敏感操作（密码修改、账户锁定）
4. 成功和失败操作都记录
5. 日志字段脱敏（密码、令牌等敏感信息不记录）

### 2.9 登录尝试表 (login_attempts)
登录尝试记录表，用于防暴力破解。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |

| client_info | JSON | - | NULL | 客户端信息 |

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 用户名索引：`INDEX idx_username_time (username, attempt_time DESC)` 用于用户失败计数
- IP索引：`INDEX idx_ip_time (ip_address, attempt_time DESC)` 用于IP限制
- 时间索引：`INDEX idx_attempt_time (attempt_time DESC)` 用于清理

**分区策略**：
- 按周分区，保留8周数据
- 自动清理过期数据

**安全规则**：
1. 同一用户名5分钟内连续失败5次锁定账户30分钟
2. 同一IP地址1小时内失败20次封锁IP24小时
3. 成功登录后清除该用户名的失败记录
4. 支持CAPTCHA触发机制

## 6. 索引和查询优化策略

### 6.1 高性能查询场景
1. **用户登录验证**：`WHERE username=? AND status='active'` → 索引: (username, status)
2. **权限检查**：`JOIN user_roles → role_permissions → permissions` → 复合索引优化
3. **会话验证**：`WHERE session_token=? AND revoked_at IS NULL AND expires_at>NOW()` → 索引: (session_token)
4. **审计日志查询**：`WHERE user_id=? AND event_time BETWEEN ? AND ?` → 索引: (user_id, event_time DESC)
5. **部门树查询**：`WHERE path LIKE '/根ID/%'` → 索引: (path)

### 6.2 分区策略
1. **审计日志表**：按月分区，保留180天
2. **登录尝试表**：按周分区，保留8周
3. **用户会话表**：按过期时间分区，自动清理
4. **冷热数据分离**：活跃数据在SSD，历史数据在HDD

### 6.3 查询优化建议
1. **避免全表扫描**：WHERE条件使用索引字段
2. **分页优化**：使用游标分页（基于时间或ID）
3. **聚合查询**：定期预计算统计结果
4. **读写分离**：审计查询走只读副本

3. **完整性**：定期审计日志完整性校验
4. **隐私保护**：敏感信息脱敏，符合GDPR

-- 用户表
CREATE TABLE users (
    id VARCHAR(36) PRIMARY KEY DEFAULT (UUID()),
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    phone VARCHAR(20) UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    real_name VARCHAR(50) NOT NULL,
    department_id VARCHAR(36),
    status ENUM('active','inactive','locked','pending') NOT NULL DEFAULT 'active',
    is_superadmin BOOLEAN NOT NULL DEFAULT FALSE,
    last_login_at DATETIME,
    failed_login_attempts INT NOT NULL DEFAULT 0,

    FOREIGN KEY (department_id) REFERENCES departments(id) ON DELETE SET NULL

-- 角色表

    id VARCHAR(36) PRIMARY KEY DEFAULT (UUID()),

    is_default BOOLEAN NOT NULL DEFAULT FALSE,

    data_scope ENUM('all','department','self','custom') NOT NULL DEFAULT 'self',

-- 用户角色关联表
