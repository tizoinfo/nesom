## 1. 设计概述

### 1.1 设计目标
用户权限管理前端模块提供安全、易用、高效的用户和权限管理界面，支持：
- **用户认证**：登录、注册、密码找回、多因素认证
- **用户管理**：用户增删改查、账户状态管理、个人信息维护
- **权限管理**：角色定义、权限分配、数据权限配置
- **组织架构**：部门树管理、人员分配、负责人设置
- **审计日志**：操作记录查询、安全事件监控、合规报告
- **安全配置**：密码策略、会话策略、安全规则配置

### 1.2 设计原则
1. **安全性**：敏感操作二次确认，权限验证，输入验证
2. **易用性**：直观的权限分配界面，批量操作支持
3. **一致性**：组件、交互、样式统一规范
4. **响应式**：全设备兼容，管理后台优先桌面端
5. **性能**：大数据量下的流畅体验，虚拟滚动
6. **可访问性**：支持键盘导航、屏幕阅读器，符合WCAG 2.1

### 1.3 技术约束
- **框架**: Vue 3.4 Composition API + `<script setup>`
- **语言**: TypeScript 5.3 (严格模式)
- **UI库**: Element Plus 2.7 + 自定义安全主题
- **状态管理**: Pinia (用户状态、权限状态)
- **路由**: Vue Router 4 + 路由守卫（权限控制）
- **HTTP客户端**: Axios + 自定义拦截器（Token刷新）
- **加密库**: crypto-js (前端加密，如MFA密钥)
- **图表**: ECharts 5 (审计统计图表)
- **WebSocket**: 原生WebSocket (实时会话管理)

### 2.3 路由守卫设计
```typescript
// 路由权限守卫
const permissionGuard: NavigationGuard = (to, from, next) => {
  const authStore = useAuthStore();
  const userStore = useUserStore();
  
  // 1. 检查是否认证
  if (!authStore.isAuthenticated) {
    if (to.meta.requiresAuth) {
      // 重定向到登录页，保存目标路由
      next({
        path: '/auth/login',
        query: { redirect: to.fullPath }
      });
      return;
    }
    next();
    return;
  }
  
  // 2. 检查权限
  if (to.meta.requiredPermissions) {
    const hasPermission = userStore.hasPermissions(to.meta.requiredPermissions);
    if (!hasPermission) {
      // 显示权限不足提示
      showPermissionDeniedModal();
      next(false);
      return;
    }
  }
  
  // 3. 检查数据权限（如需要）
  if (to.meta.requiredDataScope) {
    const hasDataScope = userStore.hasDataScope(to.meta.requiredDataScope);
    if (!hasDataScope) {
      next({ name: 'forbidden' });
      return;
    }
  }
  
  next();
};
```

#### 3.1.5 业务逻辑
```typescript
// 登录处理逻辑
const handleSubmit = async () => {
  try {
    // 表单验证
    await formRef.value?.validate();
    
    // 设置加载状态
    ui.isLoading = true;
    
    // 调用登录API
    const response = await authApi.login({
      username: form.username,
      password: form.password,
      mfa_code: form.mfaCode || undefined,
      remember_me: form.rememberMe
    });
    
    if (response.data.mfa_required) {
      // 需要MFA验证
      ui.showMfaField = true;
      emit('mfa-required', response.data);
      return;
    }
    
    // 登录成功
    authStore.setToken(response.data.access_token);
    userStore.setUser(response.data.user);
    
    emit('login-success', response.data.user);
    
    // 跳转到目标页面
    router.push(props.redirectPath || '/');
    
  } catch (error: any) {
    handleLoginError(error);
  } finally {
    ui.isLoading = false;
  }
};

// 登录错误处理
const handleLoginError = (error: ApiError) => {
  // 账户锁定错误
  if (error.code === 'ACCOUNT_LOCKED') {
    const lockTime = error.detail?.lockout_remaining || 1800;
    ui.remainingLockTime = lockTime;
    startLockTimer();
  }
  
  // 需要MFA错误
  if (error.code === 'MFA_REQUIRED') {
    ui.showMfaField = true;
    return;
  }
  
  // 需要验证码错误
  if (error.code === 'CAPTCHA_REQUIRED') {
    ui.captchaRequired = true;
    refreshCaptcha();
  }
  
  // 显示错误信息
  errors.general = error.message || '登录失败';
  
  emit('login-failed', error);
};
```

#### 3.1.6 安全特性
1. **密码字段安全**：自动清除内存中的密码字符串
2. **防暴力破解**：显示剩余锁定时间，禁用提交按钮
3. **会话安全**：登录成功后清除URL中的敏感参数
4. **MFA支持**：无缝切换用户名密码和MFA验证
5. **验证码**：触发条件后显示，防止自动化攻击

## 5. 安全特性实现

### 5.1 Token自动刷新
```typescript
// Axios响应拦截器
axiosInstance.interceptors.response.use(
  (response) => response,
  async (error) => {
    const originalRequest = error.config;
    
    // 如果是401错误且不是刷新token请求
    if (error.response?.status === 401 && !originalRequest._retry) {
      originalRequest._retry = true;
      
      try {
        // 尝试刷新token
        const newToken = await authStore.refreshToken();
        
        // 更新请求头
        originalRequest.headers.Authorization = `Bearer ${newToken}`;
        
        // 重新发送原始请求
        return axiosInstance(originalRequest);
      } catch (refreshError) {
        // 刷新失败，跳转到登录页
        authStore.clearAuth();
        router.push('/auth/login');
        return Promise.reject(refreshError);
      }
    }
    
    return Promise.reject(error);
  }
);
```
