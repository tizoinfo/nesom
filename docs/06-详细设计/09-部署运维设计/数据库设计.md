## 1. 设计概述

### 1.1 设计目标
设计NESOM系统部署和运维所需的专用数据库，支持多环境部署管理、自动化运维、监控告警、备份恢复等核心功能，包括：
- 统一管理多环境（开发、测试、生产）的部署配置和状态
- 记录完整的部署历史和变更审计
- 存储监控指标和告警数据，支持历史查询和分析
- 管理备份策略和恢复记录
- 跟踪运维工单和变更请求
- 支持容量规划和性能优化

### 1.2 设计原则
1. **环境隔离**：不同环境数据物理或逻辑隔离，保障生产安全
2. **历史可追溯**：所有运维操作完整记录，支持审计和回滚
3. **高可用性**：运维数据库自身具备高可用，避免单点故障
4. **性能优化**：时序数据分区存储，热点数据缓存优化
5. **扩展性**：支持动态添加新环境和服务类型
6. **安全性**：敏感配置加密存储，访问权限严格控制

### 1.4 数据分类和存储策略
| 数据类型 | 数据量 | 读写频率 | 存储类型 | 保留期限 |
|----------|--------|----------|----------|----------|
| 部署配置 | 小 | 低频写，中频读 | 关系型数据库 | 永久 |
| 监控指标 | 极大 | 高频写，高频读 | 时序数据库 | 30天(热), 1年(温) |
| 告警记录 | 中 | 中频写，高频读 | 关系型数据库 | 1年 |
| 备份元数据 | 小 | 低频写，低频读 | 关系型数据库 | 7年 |
| 运维工单 | 中 | 中频写，中频读 | 关系型数据库 | 3年 |
| 审计日志 | 大 | 高频写，低频读 | 时序数据库 | 180天 |
| 性能基线 | 小 | 低频写，中频读 | 关系型数据库 + 缓存 | 永久 |

## 2. 核心表结构详细设计

### 2.1 环境管理表 (environments)
管理多环境配置和状态。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID() | 环境唯一标识 |
| env_id | VARCHAR(20) | UNIQUE NOT NULL | - | 环境ID：dev/staging/prod等 |
| env_name | VARCHAR(50) | NOT NULL | - | 环境名称：开发环境/测试环境/生产环境 |
| env_type | ENUM('development','testing','staging','production','disaster_recovery') | NOT NULL | 'development' | 环境类型 |
| description | TEXT | - | NULL | 环境描述 |
| region | VARCHAR(50) | NOT NULL | 'cn-east-1' | 部署区域 |
| availability_zone | VARCHAR(50) | - | NULL | 可用区 |
| network_cidr | VARCHAR(20) | - | NULL | 网络CIDR |
| vpc_id | VARCHAR(50) | - | NULL | VPC ID |
| k8s_cluster_name | VARCHAR(100) | - | NULL | Kubernetes集群名称 |
| status | ENUM('initializing','active','degraded','maintenance','decommissioned') | NOT NULL INDEX | 'initializing' | 环境状态 |
| created_by | VARCHAR(36) | NOT NULL | - | 创建人 |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| last_health_check | DATETIME | - | NULL | 最后健康检查时间 |
| health_status | ENUM('healthy','warning','critical','unknown') | NOT NULL | 'unknown' | 健康状态 |
| tags | JSON | - | NULL | 环境标签 |
| metadata | JSON | - | NULL | 扩展元数据 |

**索引设计**:
- 主键索引: `PRIMARY KEY (id)`
- 唯一索引: `UNIQUE KEY uk_env_id (env_id)`
- 状态索引: `INDEX idx_env_type_status (env_type, status)`
- 区域索引: `INDEX idx_region (region, availability_zone)`

**业务规则**:
1. 环境ID全局唯一，格式：`{type}-{region}-{seq}`，如：prod-cn-east-1
2. 生产环境必须有灾难恢复环境配对
3. 环境状态流转：initializing → active → (degraded/maintenance) → active/decommissioned
4. 健康状态每5分钟更新一次

| environment_id | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 环境ID |
| config_version | VARCHAR(20) | NOT NULL | - | 配置版本 |
| config_type | ENUM('docker_compose','kubernetes','helm','terraform','ansible') | NOT NULL | 'kubernetes' | 配置类型 |
| config_content | JSON | NOT NULL | - | 配置内容 |

| secrets_ref | JSON | - | NULL | 密钥引用（加密存储） |
| dependencies | JSON | - | NULL | 服务依赖关系 |
| health_check_config | JSON | - | NULL | 健康检查配置 |
| scaling_config | JSON | - | NULL | 扩缩容配置 |
| resource_limits | JSON | - | NULL | 资源限制 |
| network_policies | JSON | - | NULL | 网络策略 |
| storage_config | JSON | - | NULL | 存储配置 |
| is_active | BOOLEAN | NOT NULL INDEX | TRUE | 是否激活 |
| deployed_at | DATETIME | - | NULL | 部署时间 |
| deployed_by | VARCHAR(36) | - | NULL | 部署人 |
| rollback_from | VARCHAR(50) | - | NULL | 回滚来源配置ID |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**外键约束**:
- `FOREIGN KEY (environment_id) REFERENCES environments(id) ON DELETE CASCADE`

**索引设计**:
- 主键索引: `PRIMARY KEY (id)`
- 唯一索引: `UNIQUE KEY uk_config_id (config_id)`
- 服务环境索引: `INDEX idx_service_env (service_name, environment_id, is_active)`

**业务规则**:
1. 配置ID格式：`{service_name}-{env_id}-v{version}`
2. 同一服务在同一环境只能有一个激活配置
3. 配置变更必须创建新版本，支持版本回滚
4. 敏感信息（密码、密钥）不直接存储，存储密钥引用

记录完整的部署历史，支持审计和回滚。

| config_id | VARCHAR(50) | NOT NULL INDEX | - | 配置ID |
| deployment_type | ENUM('initial','upgrade','rollback','scale','config_update') | NOT NULL | 'upgrade' | 部署类型 |
| previous_version | VARCHAR(20) | - | NULL | 前一个版本 |
| target_version | VARCHAR(20) | NOT NULL | - | 目标版本 |
| status | ENUM('pending','in_progress','succeeded','failed','cancelled','rolled_back') | NOT NULL INDEX | 'pending' | 部署状态 |
| initiated_by | VARCHAR(36) | NOT NULL | - | 发起人 |
| initiated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 发起时间 |
| started_at | DATETIME | - | NULL | 开始时间 |
| completed_at | DATETIME | - | NULL | 完成时间 |
| duration_seconds | INT | - | NULL | 持续时间（秒） |
| artifacts | JSON | - | NULL | 部署产物信息 |
| deployment_log | TEXT | - | NULL | 部署日志（压缩存储） |
| error_message | TEXT | - | NULL | 错误信息 |
| rollback_reason | TEXT | - | NULL | 回滚原因 |
| metrics | JSON | - | NULL | 部署指标（资源使用、时间等） |
| approval_id | VARCHAR(36) | - | NULL | 审批ID |
| ci_cd_pipeline_id | VARCHAR(100) | - | NULL | CI/CD流水线ID |
| git_commit_hash | VARCHAR(40) | INDEX | NULL | Git提交哈希 |
| git_branch | VARCHAR(100) | - | NULL | Git分支 |
| tags | JSON | - | NULL | 标签 |

**外键约束**:
- `FOREIGN KEY (environment_id) REFERENCES environments(id) ON DELETE CASCADE`

**索引设计**:
- 主键索引: `PRIMARY KEY (id)`
- 唯一索引: `UNIQUE KEY uk_deployment_id (deployment_id)`
- 时间范围索引: `INDEX idx_env_time (environment_id, initiated_at DESC)`
- 状态索引: `INDEX idx_service_status (service_name, status, initiated_at DESC)`
- Git索引: `INDEX idx_git_commit (git_commit_hash)`

**分区策略**:
- 按月分区: `PARTITION BY RANGE (TO_DAYS(initiated_at))`
- 保留24个月数据，历史数据归档

**业务规则**:
1. 部署ID格式：`DEP-{env_id}-{timestamp}-{random6}`
2. 部署状态流转：pending → in_progress → succeeded/failed/cancelled → (可选)rolled_back
3. 失败部署自动重试最多3次
4. 生产环境部署需要审批记录

| rule_type | ENUM('metric_threshold','anomaly_detection','log_pattern','health_check') | NOT NULL | 'metric_threshold' | 规则类型 |
| metric_name | VARCHAR(100) | - | NULL | 指标名称 |

| severity | ENUM('info','warning','critical') | NOT NULL | 'warning' | 严重程度 |

| escalation_policy | JSON | - | NULL | 升级策略 |
| enabled | BOOLEAN | NOT NULL INDEX | TRUE | 是否启用 |

| created_by | VARCHAR(36) | NOT NULL | - | 创建人 |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| last_triggered | DATETIME | - | NULL | 最后触发时间 |

**外键约束**:
- `FOREIGN KEY (environment_id) REFERENCES environments(id) ON DELETE CASCADE`

**索引设计**:
- 主键索引: `PRIMARY KEY (id)`
- 唯一索引: `UNIQUE KEY uk_rule_id (rule_id)`
- 环境服务索引: `INDEX idx_env_service (environment_id, service_name, enabled)`
- 类型索引: `INDEX idx_rule_type (rule_type, severity)`

**业务规则**:
1. 规则ID格式：`{rule_type}-{metric_name}-{env_id?}-{seq}`
2. 优先级：服务级规则 > 环境级规则 > 全局规则
3. 相同条件持续达到for_duration才触发告警
4. 告警静默期间不发送通知，但仍记录触发

| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引设计**:
- 主键索引: `PRIMARY KEY (id)`
- 唯一索引: `UNIQUE KEY uk_alert_id (alert_id)`
- 时间范围索引: `INDEX idx_time_range (start_time DESC, end_time)`
- 状态索引: `INDEX idx_status_severity (status, severity, start_time DESC)`
- 环境服务索引: `INDEX idx_env_service_status (environment_id, service_name, status)`

**分区策略**:
- 按月分区: `PARTITION BY RANGE (TO_DAYS(start_time))`
- 保留12个月数据，历史数据归档

**业务规则**:
1. 告警ID格式：`ALERT-{timestamp_ms}-{random8}`
2. 告警状态流转：firing → acknowledged → resolved 或 firing → resolved
3. 相同标签的告警自动分组
4. 已解决的告警保留历史记录，支持分析

1. **数据压缩**: 历史数据启用压缩存储
2. **分区管理**: 按时序分区，自动管理分区
3. **存储引擎**: 根据访问模式选择合适存储引擎
4. **连接池**: 数据库连接池化，避免连接风暴

4. **故障恢复**: 原主库恢复后作为新备库加入

```sql
-- 环境管理表
CREATE TABLE environments (
    id VARCHAR(36) PRIMARY KEY DEFAULT (UUID()),
    env_id VARCHAR(20) UNIQUE NOT NULL,
    env_name VARCHAR(50) NOT NULL,
    env_type ENUM('development','testing','staging','production','disaster_recovery') NOT NULL DEFAULT 'development',
    description TEXT,
    region VARCHAR(50) NOT NULL DEFAULT 'cn-east-1',
    availability_zone VARCHAR(50),
    network_cidr VARCHAR(20),
    vpc_id VARCHAR(50),
    k8s_cluster_name VARCHAR(100),
    status ENUM('initializing','active','degraded','maintenance','decommissioned') NOT NULL DEFAULT 'initializing',
    created_by VARCHAR(36) NOT NULL,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    last_health_check DATETIME,
    health_status ENUM('healthy','warning','critical','unknown') NOT NULL DEFAULT 'unknown',
    tags JSON,
    metadata JSON,
    INDEX idx_env_type_status (env_type, status),
    INDEX idx_region (region, availability_zone)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 部署配置表
CREATE TABLE deployment_configs (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    config_id VARCHAR(50) UNIQUE NOT NULL,
    service_name VARCHAR(100) NOT NULL,
    environment_id VARCHAR(36) NOT NULL,
    config_version VARCHAR(20) NOT NULL,
    config_type ENUM('docker_compose','kubernetes','helm','terraform','ansible') NOT NULL DEFAULT 'kubernetes',
    config_content JSON NOT NULL,
    variables JSON,
    secrets_ref JSON,
    dependencies JSON,
    health_check_config JSON,
    scaling_config JSON,
    resource_limits JSON,
    network_policies JSON,
    storage_config JSON,
    is_active BOOLEAN NOT NULL DEFAULT TRUE,
    deployed_at DATETIME,
    deployed_by VARCHAR(36),
    rollback_from VARCHAR(50),
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_service_env (service_name, environment_id, is_active),
    INDEX idx_config_version (config_version DESC),
    FOREIGN KEY (environment_id) REFERENCES environments(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 10.3 维护脚本
```bash
#!/bin/bash
# 数据库维护脚本

# 1. 备份数据库
mysqldump -h $DB_HOST -u $DB_USER -p$DB_PASS operations_db > /backup/operations_$(date +%Y%m%d).sql

# 2. 优化表
mysql -h $DB_HOST -u $DB_USER -p$DB_PASS operations_db -e "OPTIMIZE TABLE deployment_history, alert_records, backup_records;"

# 3. 清理过期数据
mysql -h $DB_HOST -u $DB_USER -p$DB_PASS operations_db -e "
DELETE FROM deployment_history WHERE initiated_at < DATE_SUB(NOW(), INTERVAL 2 YEAR);
DELETE FROM alert_records WHERE start_time < DATE_SUB(NOW(), INTERVAL 1 YEAR);
"

# 4. 更新统计信息
mysql -h $DB_HOST -u $DB_USER -p$DB_PASS operations_db -e "ANALYZE TABLE;"
```
