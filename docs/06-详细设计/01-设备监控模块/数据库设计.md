## 1. 设计概述

### 1.1 设计目标
设备监控模块负责管理新能源设备全生命周期数据，包括：
- 设备基础信息管理（分类、属性、位置）
- 设备实时状态监控与数据采集
- 设备运行指标存储与历史查询
- 设备异常检测与告警管理
- 设备维护计划与健康评估

### 1.2 设计原则
1. **性能优先**：高频查询字段添加索引，历史数据分区存储
2. **扩展性**：支持多设备类型，JSON字段存储可变参数
3. **完整性**：外键约束保障数据一致性
4. **可维护性**：清晰的表关系和注释
5. **监控友好**：支持实时查询和聚合分析

### 1.3 与其他模块关系
- **依赖模块**：用户权限管理（设备负责人、操作人员）
- **关联模块**：工单管理（告警转工单）、巡检管理（设备巡检）
- **数据提供**：报表统计（设备运行数据）

## 2. 表结构详细设计

### 2.1 设备类型表 (device_types)
设备分类体系，支持树形结构，定义设备参数模板。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID() | 主键 |
| type_code | VARCHAR(50) | UNIQUE NOT NULL | - | 类型编码，唯一标识 |
| type_name | VARCHAR(100) | NOT NULL | - | 类型名称 |
| sort_order | INT | NOT NULL | 0 | 显示排序 |
| parent_id | VARCHAR(36) | FOREIGN KEY | NULL | 父类型ID，支持树形结构 |
| category | VARCHAR(50) | INDEX | NULL | 设备大类：光伏、风电、储能等 |
| sub_category | VARCHAR(50) | - | NULL | 设备子类 |
| parameter_template | JSON | - | NULL | 参数模板，定义设备参数结构 |
| maintenance_template | JSON | - | NULL | 维护模板，定义维护项和周期 |
| description | TEXT | - | NULL | 类型描述 |
| icon | VARCHAR(255) | - | NULL | 类型图标路径 |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 唯一索引：`UNIQUE KEY uk_type_code (type_code)`
- 查询索引：`INDEX idx_category (category)`, `INDEX idx_parent_id (parent_id)`

**业务规则**：
1. 类型编码全局唯一，用于程序识别
2. 树形结构最大深度不超过5层
3. parameter_template为JSON格式，定义设备参数验证规则
4. maintenance_template定义预防性维护计划

### 2.2 设备表 (devices)
设备核心信息表，存储设备静态属性和运行状态。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID() | 主键 |
| station_id | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 所属场站ID |
| device_type_id | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 设备类型ID |
| device_code | VARCHAR(50) | NOT NULL | - | 设备编码，场站内唯一 |
| device_name | VARCHAR(100) | NOT NULL | - | 设备名称 |
| status | VARCHAR(20) | NOT NULL INDEX | 'offline' | 设备状态：online/offline/fault/maintenance/testing |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| area_id | VARCHAR(36) | - | NULL | 区域ID（物理分区） |
| manufacturer | VARCHAR(100) | INDEX | NULL | 制造商 |
| model | VARCHAR(100) | - | NULL | 型号 |
| serial_number | VARCHAR(100) | - | NULL | 序列号 |
| rated_power | DECIMAL(10,2) | - | NULL | 额定功率(kW) |
| rated_voltage | DECIMAL(8,2) | - | NULL | 额定电压(V) |
| rated_current | DECIMAL(8,2) | - | NULL | 额定电流(A) |
| parameters | JSON | - | NULL | 设备参数，动态扩展 |
| installation_date | DATE | - | NULL | 安装日期 |
| commissioning_date | DATE | - | NULL | 投运日期 |
| warranty_period | INT | - | NULL | 保修期(月) |
| warranty_expiry | DATE | - | NULL | 保修到期日 |
| health_score | INT | - | NULL | 健康评分(0-100) |
| last_maintenance_date | DATE | - | NULL | 上次维护日期 |
| next_maintenance_date | DATE | - | NULL | 下次计划维护日期 |
| location_description | TEXT | - | NULL | 位置描述 |
| longitude | DECIMAL(10,6) | - | NULL | 经度 |
| latitude | DECIMAL(10,6) | - | NULL | 纬度 |
| altitude | DECIMAL(8,2) | - | NULL | 海拔(m) |
| description | TEXT | - | NULL | 设备描述 |
| images | JSON | - | NULL | 设备图片列表 |
| documents | JSON | - | NULL | 相关文档 |
| qr_code | VARCHAR(255) | - | NULL | 二维码内容 |
| responsible_person_id | VARCHAR(36) | - | NULL | 负责人用户ID |
| responsible_person_name | VARCHAR(100) | - | NULL | 负责人姓名 |
| last_heartbeat | DATETIME | - | NULL | 最后心跳时间 |
| data_collection_status | VARCHAR(20) | - | 'disabled' | 数据采集状态：enabled/disabled/error |
| data_collection_config | JSON | - | NULL | 数据采集配置 |

**外键约束**：
- `FOREIGN KEY (station_id) REFERENCES stations(id) ON DELETE CASCADE`
- `FOREIGN KEY (device_type_id) REFERENCES device_types(id) ON DELETE RESTRICT`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 唯一约束：`UNIQUE KEY uk_station_device_code (station_id, device_code)`（场站内设备编码唯一）
- 查询索引：`INDEX idx_status (status)`, `INDEX idx_device_type (device_type_id)`, `INDEX idx_station (station_id)`, `INDEX idx_manufacturer (manufacturer)`
- 组合索引：`INDEX idx_station_status (station_id, status)` 用于场站设备状态统计

**业务规则**：
1. device_code在场站内必须唯一，格式：`STATION_CODE-DEVICE_TYPE-SEQ`
2. status状态流转：offline → online → (fault/maintenance/testing) → online/offline
3. last_heartbeat超过5分钟未更新，自动标记为offline
4. health_score根据设备运行数据自动计算
5. data_collection_config定义采集频率、协议、参数映射

### 2.3 设备指标表 (device_metrics)
设备运行指标时序数据表，高频写入，按时间分区。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | BIGINT | PRIMARY KEY AUTO_INCREMENT | - | 自增主键 |
| device_id | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 设备ID |
| metric_type | VARCHAR(50) | NOT NULL INDEX | - | 指标类型：voltage/current/power/temperature/humidity等 |
| metric_value | DECIMAL(12,4) | NOT NULL | - | 指标值 |
| metric_unit | VARCHAR(20) | NOT NULL | - | 指标单位：V/A/kW/℃/%等 |
| quality | TINYINT | - | 100 | 数据质量(0-100) |
| collected_at | DATETIME(3) | NOT NULL INDEX | - | 采集时间，毫秒精度 |
| received_at | DATETIME(3) | NOT NULL | CURRENT_TIMESTAMP(3) | 接收时间 |
| source | VARCHAR(50) | - | 'direct' | 数据来源：direct/calculated/imported |
| tags | JSON | - | NULL | 标签数据，扩展属性 |

**外键约束**：
- `FOREIGN KEY (device_id) REFERENCES devices(id) ON DELETE CASCADE`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 查询索引：`INDEX idx_device_collected (device_id, collected_at DESC)` 主查询路径
- 时间索引：`INDEX idx_collected_at (collected_at)` 时间范围查询
- 类型索引：`INDEX idx_metric_type (metric_type)` 按指标类型查询

**分区策略**：
- 按时间范围分区：`PARTITION BY RANGE (TO_DAYS(collected_at))`
- 每月一个分区，保留24个月数据
- 历史数据自动归档到历史表

**业务规则**：
1. 高频采集数据（秒级/分钟级）存储在此表
2. 每日凌晨生成日聚合数据到device_metrics_daily表
3. 数据质量<60的数据标记为可疑，需人工确认
4. collected_at为设备实际采集时间，received_at为系统接收时间

### 2.4 设备告警表 (device_alerts)
设备异常告警记录，支持多级告警和确认流程。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | BIGINT | PRIMARY KEY AUTO_INCREMENT | - | 自增主键 |
| device_id | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 设备ID |
| alert_code | VARCHAR(50) | NOT NULL INDEX | - | 告警编码，唯一标识 |
| alert_type | VARCHAR(50) | NOT NULL | - | 告警类型：threshold/exceedance/fault/communication等 |
| alert_level | ENUM('info','warning','error','critical') | NOT NULL INDEX | 'warning' | 告警级别 |
| alert_title | VARCHAR(200) | NOT NULL | - | 告警标题 |
| alert_message | TEXT | NOT NULL | - | 告警详情 |
| alert_data | JSON | - | NULL | 告警相关数据 |
| trigger_value | DECIMAL(12,4) | - | NULL | 触发值 |
| threshold_value | DECIMAL(12,4) | - | NULL | 阈值 |
| start_time | DATETIME(3) | NOT NULL INDEX | - | 告警开始时间 |
| end_time | DATETIME(3) | - | NULL | 告警结束时间 |
| acknowledged_at | DATETIME | - | NULL | 确认时间 |
| acknowledged_by | VARCHAR(36) | - | NULL | 确认人ID |
| acknowledged_by_name | VARCHAR(100) | - | NULL | 确认人姓名 |
| resolved_at | DATETIME | - | NULL | 解决时间 |
| resolved_by | VARCHAR(36) | - | NULL | 解决人ID |
| resolved_by_name | VARCHAR(100) | - | NULL | 解决人姓名 |
| resolution_notes | TEXT | - | NULL | 解决说明 |
| status | ENUM('active','acknowledged','resolved','closed') | NOT NULL INDEX | 'active' | 告警状态 |
| related_work_order_id | VARCHAR(36) | - | NULL | 关联工单ID |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**外键约束**：
- `FOREIGN KEY (device_id) REFERENCES devices(id) ON DELETE CASCADE`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 查询索引：`INDEX idx_device_status (device_id, status)`, `INDEX idx_alert_level (alert_level)`, `INDEX idx_start_time (start_time DESC)`
- 告警编码：`UNIQUE KEY uk_alert_code (alert_code)`（防止重复告警）

**业务规则**：
1. 告警编码格式：`ALERT-{设备编码}-{时间戳}-{序列号}`
2. 告警状态流转：active → acknowledged → resolved → closed
3. 未确认的critical级别告警需在5分钟内通知
4. 告警持续超过24小时自动升级级别
5. resolved状态需填写resolution_notes

### 2.5 设备阈值表 (device_thresholds)
设备告警阈值配置，支持设备级、类型级、全局级阈值。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |

| device_id | VARCHAR(36) | FOREIGN KEY | NULL | 设备ID（NULL表示类型级或全局） |
| device_type_id | VARCHAR(36) | FOREIGN KEY | NULL | 设备类型ID（NULL表示全局） |
| metric_type | VARCHAR(50) | NOT NULL | - | 指标类型 |
| threshold_type | ENUM('min','max','range','rate_of_change') | NOT NULL | 'max' | 阈值类型 |
| warning_value | DECIMAL(12,4) | - | NULL | 警告阈值 |
| error_value | DECIMAL(12,4) | - | NULL | 错误阈值 |
| critical_value | DECIMAL(12,4) | - | NULL | 严重阈值 |
| custom_expression | TEXT | - | NULL | 自定义表达式 |
| is_enabled | BOOLEAN | NOT NULL | TRUE | 是否启用 |
|生效开始时间 | DATETIME | - | NULL | 生效开始时间 |
|生效结束时间 | DATETIME | - | NULL | 生效结束时间 |
| description | TEXT | - | NULL | 阈值描述 |
| created_by | VARCHAR(36) | NOT NULL | - | 创建人 |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**外键约束**：
- `FOREIGN KEY (device_id) REFERENCES devices(id) ON DELETE CASCADE`
- `FOREIGN KEY (device_type_id) REFERENCES device_types(id) ON DELETE CASCADE`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 查询索引：`INDEX idx_device_metric (device_id, metric_type)`，`INDEX idx_type_metric (device_type_id, metric_type)`
- 唯一约束：`UNIQUE KEY uk_device_metric (device_id, metric_type, threshold_type)`（设备级唯一）

**业务规则**：
1. 阈值优先级：设备级 > 类型级 > 全局级
2. 支持时间范围生效（如：夏季高温阈值）
3. custom_expression支持JavaScript表达式，可引用历史数据
4. 阈值修改需记录审计日志

| operator_id | VARCHAR(36) | - | NULL | 操作人ID（手动操作时） |

2. 事件数据包含变更前后的值
3. 失败的事件需记录详细错误信息
4. 事件保留2年，支持审计追溯

### 3.2 关系说明
1. **设备类型-设备 (1:N)**：一个设备类型对应多个设备，设备必须属于一个类型
2. **场站-设备 (1:N)**：一个场站包含多个设备，设备必须属于一个场站
3. **设备-指标数据 (1:N)**：一个设备有多个时序指标数据点
4. **设备-告警 (1:N)**：一个设备可产生多个告警
5. **设备-阈值 (1:N)**：一个设备可配置多个指标阈值
6. **设备-事件 (1:N)**：一个设备有多个事件记录
7. **设备-维护记录 (1:N)**：一个设备有多次维护记录

## 4. 数据字典

### 4.1 设备状态枚举 (devices.status)
| 值 | 描述 | 业务含义 |

## 6. 索引和查询优化策略

### 6.1 高性能查询场景
1. **设备状态看板**：`WHERE station_id=? AND status IN (?)` → 索引: (station_id, status)
2. **实时数据查询**：`WHERE device_id=? AND collected_at BETWEEN ? AND ?` → 索引: (device_id, collected_at DESC)
3. **告警监控**：`WHERE status='active' AND alert_level IN ('error','critical')` → 索引: (status, alert_level, start_time)
4. **设备搜索**：`WHERE device_name LIKE ? OR device_code LIKE ?` → 全文索引

### 6.2 分区策略
1. **设备指标表**：按月分区，自动创建下月分区
2. **历史数据归档**：24个月前数据移至历史表
3. **冷热数据分离**：近3月数据在SSD，历史数据在HDD

### 6.3 查询优化建议
1. **避免全表扫描**：WHERE条件使用索引字段
2. **分页优化**：使用游标分页而非OFFSET
3. **聚合查询**：定期预计算聚合结果
4. **读写分离**：报表查询走只读副本
