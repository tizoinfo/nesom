## 1. 安全设计概述

### 1.1 安全目标
设备监控模块作为NESOM系统的核心，处理关键设备数据和控制系统，必须满足以下安全目标：

1. **机密性**: 防止未授权访问设备数据和控制指令
2. **完整性**: 确保设备数据和配置不被篡改
3. **可用性**: 保障监控系统7×24小时可靠运行
4. **可审计性**: 所有操作可追溯、可审计
5. **合规性**: 符合等保2.0三级、网络安全法等要求

### 1.2 安全威胁模型

#### 1.2.1 威胁识别
| 威胁类型 | 威胁描述 | 影响等级 | 可能性 |
|----------|----------|----------|--------|
| 未授权访问 | 攻击者获取设备控制权限 | 严重 | 中 |
| 数据泄露 | 设备运行数据、配置信息泄露 | 高 | 中 |
| 拒绝服务 | 攻击导致监控系统不可用 | 高 | 高 |
| 数据篡改 | 恶意修改设备数据或阈值 | 严重 | 中 |
| 供应链攻击 | 通过第三方组件入侵系统 | 中 | 低 |
| 内部威胁 | 内部人员滥用权限 | 高 | 中 |

#### 1.2.2 攻击面分析
1. **Web界面**: API端点、WebSocket连接、文件上传
2. **数据采集**: 设备通信协议、数据解析服务
3. **数据库**: SQL注入、未授权访问
4. **消息队列**: 消息篡改、重放攻击
5. **文件存储**: 文件上传漏洞、路径遍历
6. **第三方集成**: 地图服务、短信/邮件服务

### 1.3 安全设计原则
1. **最小权限原则**: 用户和系统组件仅拥有必要权限
2. **纵深防御**: 多层安全防护，单一防线失效不影响整体
3. **默认安全**: 默认配置即为安全配置
4. **安全左移**: 安全考虑贯穿开发全生命周期
5. **零信任**: 不信任任何内部或外部实体
6. **隐私保护**: 用户和设备数据最小化收集

```python
# JWT令牌生成和验证
class JWTAuthService:
    def __init__(self):
        # 使用非对称加密
        self.private_key = load_private_key('certs/private.pem')
        self.public_key = load_public_key('certs/public.pem')
    
    def create_access_token(self, user: User) -> str:
        """创建访问令牌"""
        now = datetime.utcnow()
        payload = {
            'sub': user.id,
            'username': user.username,
            'role': user.role,
            'permissions': user.permissions,
            'iat': now,
            'exp': now + timedelta(minutes=15),  # 短有效期
            'jti': str(uuid.uuid4()),  # 唯一标识，防重放
            'iss': 'nesom-auth',       # 签发者
            'aud': 'nesom-api',        # 接收者
        }
        
        # 添加设备上下文（如适用）
        if user.current_device_id:
            payload['device_id'] = user.current_device_id
        
        return jwt.encode(payload, self.private_key, algorithm='RS256')
    
    def verify_token(self, token: str) -> Dict:
        """验证JWT令牌"""
        try:
            payload = jwt.decode(
                token,
                self.public_key,
                algorithms=['RS256'],
                issuer='nesom-auth',
                audience='nesom-api',
                options={'require': ['exp', 'iat', 'iss', 'aud']}
            )
            
            # 检查令牌是否在黑名单中
            if self.is_token_revoked(payload['jti']):
                raise TokenRevokedError('令牌已撤销')
            
            return payload
        except jwt.ExpiredSignatureError:
            raise TokenExpiredError('令牌已过期')
        except jwt.InvalidTokenError as e:
            raise InvalidTokenError(f'无效令牌: {str(e)}')
```

### 2.2 访问控制设计

#### 2.2.1 基于角色的访问控制 (RBAC)
```python
# 权限模型定义
class Permission:
    """权限定义"""
    def __init__(self, resource: str, action: str, conditions: List[Callable] = None):
        self.resource = resource  # 资源类型: device, alert, config等
        self.action = action      # 操作: create, read, update, delete, control
        self.conditions = conditions or []  # 动态条件
        
    def check(self, user: User, resource: Any = None) -> bool:
        """检查权限"""
        # 检查静态权限
        if not user.has_permission(f"{self.resource}:{self.action}"):
            return False
        
        # 检查动态条件
        for condition in self.conditions:
            if not condition(user, resource):
                return False
        
        return True


# 权限条件示例
def same_station_condition(user: User, device: Device) -> bool:
    """用户和设备必须在同一场站"""
    return user.station_id == device.station_id


def device_owner_condition(user: User, device: Device) -> bool:
    """用户必须是设备负责人"""
    return user.id == device.responsible_person_id


# 权限定义
PERMISSIONS = {
    # 设备相关权限
    'device:read': Permission('device', 'read'),
    'device:create': Permission('device', 'create'),
    'device:update': Permission('device', 'update', [same_station_condition]),
    'device:delete': Permission('device', 'delete', [same_station_condition]),
    'device:control': Permission('device', 'control', [
        same_station_condition,
        device_owner_condition  # 控制需要负责人权限
    ]),
    
    # 告警相关权限
    'alert:read': Permission('alert', 'read'),
    'alert:acknowledge': Permission('alert', 'update', [same_station_condition]),
    'alert:resolve': Permission('alert', 'update', [
        same_station_condition,
        lambda u, a: u.role in ['engineer', 'admin']  # 需要工程师以上权限
    ]),
}
```

```python
@app.middleware("http")
async def security_middleware(request: Request, call_next):
    # 1. 请求ID跟踪
    request_id = request.headers.get('X-Request-ID', str(uuid.uuid4()))
    
    # 2. 安全检查
    security_checks = [
        check_rate_limit(request),      # 速率限制
        check_sql_injection(request),   # SQL注入检查
        check_xss(request),             # XSS检查
        check_csrf(request),            # CSRF检查（如适用）
    ]
    
    for check in security_checks:
        if not check[0]:
            return JSONResponse(
                status_code=400,
                content={"error": check[1]}
            )
    
    # 3. 添加安全头
    response = await call_next(request)
    
    response.headers['X-Request-ID'] = request_id
    response.headers['X-Content-Type-Options'] = 'nosniff'
    response.headers['X-Frame-Options'] = 'DENY'
    response.headers['X-XSS-Protection'] = '1; mode=block'
    response.headers['Strict-Transport-Security'] = 'max-age=31536000; includeSubDomains'
    response.headers['Content-Security-Policy'] = "default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval'; style-src 'self' 'unsafe-inline';"
    
    return response
```

#### 2.3.3 速率限制
```python
# 基于令牌桶的速率限制
class RateLimiter:
    def __init__(self):
        # 按用户/IP区分
        self.limits = {
            'anonymous': {'window': 60, 'max': 60},      # 匿名用户：60次/分钟
            'authenticated': {'window': 60, 'max': 300}, # 认证用户：300次/分钟
            'api_key': {'window': 60, 'max': 1000},     # API密钥：1000次/分钟
        }
        
        # 敏感端点额外限制
        self.endpoint_limits = {
            '/api/v1/devices/{id}/control': {'window': 60, 'max': 10},  # 控制：10次/分钟
            '/api/v1/alerts/batch': {'window': 300, 'max': 5},         # 批量操作：5次/5分钟
        }
    
    async def check_limit(self, request: Request, user: Optional[User] = None):
        """检查速率限制"""
        identifier = self._get_identifier(request, user)
        endpoint = request.url.path
        
        # 获取限制配置
        limit_config = self.limits.get(identifier['type'], self.limits['anonymous'])
        if endpoint in self.endpoint_limits:
            # 使用更严格的限制
            endpoint_config = self.endpoint_limits[endpoint]
            limit_config = {
                'window': min(limit_config['window'], endpoint_config['window']),
                'max': min(limit_config['max'], endpoint_config['max']),
            }
        
        # 检查是否超限
        key = f"rate_limit:{identifier['value']}:{endpoint}"
        current = await redis.incr(key)
        
        if current == 1:
            await redis.expire(key, limit_config['window'])
        
        if current > limit_config['max']:
            raise RateLimitExceededError(
                f"请求过于频繁，请{limit_config['window']}秒后再试"
            )
```

```javascript
// WebSocket连接安全
class SecureWebSocket {
  constructor(url, options = {}) {
    // 强制使用WSS
    if (!url.startsWith('wss://')) {
      throw new Error('WebSocket必须使用WSS协议');
    }
    
    // 添加认证令牌
    const token = localStorage.getItem('access_token');
    const secureUrl = `${url}?token=${encodeURIComponent(token)}`;
    
    // 创建连接
    this.ws = new WebSocket(secureUrl);
    
    // 设置超时和重连
    this.timeout = options.timeout || 30000;
    this.maxRetries = options.maxRetries || 3;
    this.retryCount = 0;
    
    // 心跳检测
    this.heartbeatInterval = setInterval(() => {
      if (this.ws.readyState === WebSocket.OPEN) {
        this.ws.send(JSON.stringify({ type: 'ping' }));
      }
    }, 30000);
  }
  
  // 发送加密数据（如需要）
  sendEncrypted(data) {
    const encrypted = this.encryptData(data);
    this.ws.send(encrypted);
  }
  
  // AES-GCM加密
  encryptData(data) {
    // 使用存储在安全存储中的密钥
    const key = window.crypto.subtle.importKey(
      'raw',
      this.getEncryptionKey(),
      { name: 'AES-GCM' },
      false,
      ['encrypt']
    );
    
    const iv = window.crypto.getRandomValues(new Uint8Array(12));
    const encoded = new TextEncoder().encode(JSON.stringify(data));
    
    return window.crypto.subtle.encrypt(
      { name: 'AES-GCM', iv },
      key,
      encoded
    );
  }
}
```

```python
# 使用AWS KMS加密敏感数据
class SensitiveDataEncryptor:
    def __init__(self):
        self.kms_client = boto3.client('kms', region_name='cn-north-1')
        self.key_id = 'alias/nesom-device-credentials'
    
    def encrypt_field(self, plaintext: str, context: Dict = None) -> Dict:
        """加密字段"""
        response = self.kms_client.encrypt(
            KeyId=self.key_id,
            Plaintext=plaintext.encode('utf-8'),
            EncryptionContext=context or {}
        )
        
        return {
            'ciphertext': base64.b64encode(response['CiphertextBlob']).decode('utf-8'),
            'key_id': response['KeyId'],
            'encryption_context': context,
        }
    
    def decrypt_field(self, encrypted_data: Dict) -> str:
        """解密字段"""
        ciphertext = base64.b64decode(encrypted_data['ciphertext'])
        
        response = self.kms_client.decrypt(
            CiphertextBlob=ciphertext,
            EncryptionContext=encrypted_data.get('encryption_context', {})
        )
        
        return response['Plaintext'].decode('utf-8')
```

```python
class DataRetentionManager:
    def __init__(self):
        self.retention_policies = {
            # 实时数据：保留30天
            'device_metrics': {
                'retention_days': 30,
                'archive_before_delete': True,
                'sharding': 'monthly',
            },
            # 告警数据：保留2年
            'device_alerts': {
                'retention_days': 730,
                'archive_before_delete': False,
                'sharding': 'quarterly',
            },
            # 操作日志：保留180天（合规要求）
            'audit_logs': {
                'retention_days': 180,
                'archive_before_delete': True,
                'sharding': 'monthly',
                'immutable': True,  # 不可修改
            },
            # 临时文件：保留7天
            'temp_files': {
                'retention_days': 7,
                'archive_before_delete': False,
            }
        }
    
    async def execute_retention(self):
        """执行数据保留策略"""
        for table_name, policy in self.retention_policies.items():
            cutoff_date = datetime.now() - timedelta(days=policy['retention_days'])
            
            # 归档数据（如配置）
            if policy.get('archive_before_delete', False):
                await self.archive_data(table_name, cutoff_date)
            
            # 安全删除
            await self.safe_delete(table_name, cutoff_date)
    
    async def safe_delete(self, table_name: str, cutoff_date: datetime):
        """安全删除数据（多次覆写）"""
        # 1. 标记为待删除
        await db.execute(f"""
            UPDATE {table_name} 
            SET deletion_status = 'pending'
            WHERE created_at < %s 
            AND deletion_status IS NULL
        """, cutoff_date)
        
        # 2. 记录删除审计
        deleted_count = await db.execute(f"""
            SELECT COUNT(*) FROM {table_name}
            WHERE deletion_status = 'pending'
        """)
        
        await audit.log_deletion(
            table=table_name,
            count=deleted_count,
            cutoff_date=cutoff_date,
            operator='system'
        )
        
        # 3. 实际删除（小批量，避免锁表）
        batch_size = 1000
        while True:
            rows = await db.execute(f"""
                DELETE FROM {table_name}
                WHERE deletion_status = 'pending'
                LIMIT %s
            """, batch_size)
            
            if rows == 0:
                break
            
            await asyncio.sleep(1)  # 避免对生产系统影响
```

```python
class DeviceTLSAuth:

        # 设备证书存储
        self.device_certs = {}  # device_id -> certificate
        
    def authenticate_device(self, client_cert: bytes, device_id: str) -> bool:
        """验证设备证书"""
        try:
            # 验证证书链
            cert = x509.load_pem_x509_certificate(client_cert)
            
            # 检查证书是否由CA签发
            if not self.verify_certificate_chain(cert):
                return False
            
            # 检查设备ID匹配
            cert_device_id = self.extract_device_id(cert)
            if cert_device_id != device_id:
                return False
            
            # 检查证书是否在撤销列表
            if self.is_certificate_revoked(cert):
                return False
            
            # 验证通过，缓存证书
            self.device_certs[device_id] = cert
            return True
            
        except Exception as e:
            logger.error(f"设备证书验证失败: {e}")
            return False
    
    def get_device_certificate(self, device_id: str) -> Optional[x509.Certificate]:
        """获取设备证书"""
```

```python
# 数据完整性保护
class DataIntegrityVerifier:
    def __init__(self):
        self.hmac_key = load_hmac_key()
    
    def sign_data(self, device_id: str, data: Dict, timestamp: datetime) -> str:
        """为数据生成数字签名"""
        # 构造签名字符串
        sign_string = f"{device_id}:{timestamp.isoformat()}:{json.dumps(data, sort_keys=True)}"
        
        # 计算HMAC
        hmac_obj = hmac.new(
            key=self.hmac_key,
            msg=sign_string.encode('utf-8'),
            digestmod='sha256'
        )
        
        return hmac_obj.hexdigest()
    
    def verify_data(self, device_id: str, data: Dict, timestamp: datetime, signature: str) -> bool:
        """验证数据完整性"""
        # 重新计算签名
        expected_signature = self.sign_data(device_id, data, timestamp)
        
        # 使用恒定时间比较防止时序攻击
        return hmac.compare_digest(expected_signature, signature)
```

**响应措施**:
1. **立即隔离**: 断开设备网络连接，切换为本地控制
2. **撤销凭证**: 立即撤销设备所有访问凭证
3. **日志分析**: 分析控制日志，确定攻击路径
4. **系统检查**: 检查相关系统和账户是否受影响
5. **恢复控制**: 重新颁发安全凭证，恢复控制
6. **加固防护**: 加强设备认证和访问控制

#### 7.2.2 数据泄露事件
**场景**: 设备运行数据或配置信息泄露

**响应措施**:
1. **确定范围**: 确定泄露的数据类型和数量
2. **控制扩散**: 删除泄露数据，通知相关方
3. **调查原因**: 分析泄露途径（内部/外部）
4. **修复漏洞**: 修补导致泄露的安全漏洞
5. **合规报告**: 如涉及个人数据，依法报告
6. **监控预防**: 加强数据访问监控和防护

```python
class DataClassification:
    def __init__(self):
        self.classification_rules = {
            'device_control': {
                'level': '特级',
                'sensitivity': '高',
                'protection': '加密存储、访问控制、操作审计',
            },
            'user_pii': {
                'level': '一级',
                'sensitivity': '高', 
                'protection': '加密存储、脱敏展示、访问审批',
            },
            'device_metrics': {
                'level': '二级',
                'sensitivity': '中',
                'protection': '访问控制、导出限制',
            },
            'system_logs': {
                'level': '三级',
                'sensitivity': '低',
                'protection': '访问日志、防篡改',
            },
        }
    
    def classify_data(self, data_type: str, content: Dict) -> Dict:
        """数据分类分级"""
        classification = self.classification_rules.get(data_type, {
            'level': '三级',
            'sensitivity': '低',
            'protection': '基础防护',
        })
        
        # 动态调整（根据内容）
        if data_type == 'device_metrics' and self.contains_sensitive_metrics(content):
            classification = {
                'level': '一级',
                'sensitivity': '高',
                'protection': '加密存储、脱敏展示',
            }
        
        return classification
```
