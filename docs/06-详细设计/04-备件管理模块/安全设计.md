## 1. 安全设计概述

### 1.1 安全目标
备件管理模块作为NESOM系统的物资管理核心，处理备件全生命周期数据，必须满足以下安全目标：

1. **数据机密性**: 防止未授权访问备件档案、库存数据、采购价格、供应商信息
2. **操作完整性**: 确保库存事务、成本核算、采购流程数据不被篡改
3. **业务可用性**: 保障备件管理流程7×24小时可靠运行，防止库存超卖
4. **操作可追溯**: 所有库存操作可审计、可追溯、不可抵赖
5. **合规性**: 符合等保2.0三级、数据安全法、采购法规要求

### 1.2 安全威胁模型

#### 1.2.1 威胁识别
| 威胁类型 | 威胁描述 | 影响等级 | 可能性 |
|----------|----------|----------|--------|
| 库存数据泄露 | 攻击者获取库存详情、采购价格、成本信息 | 严重 | 中 |
| 库存操作篡改 | 恶意修改库存数量、采购订单、成本数据 | 严重 | 中 |
| 库存超卖攻击 | 并发操作导致库存超卖，影响业务连续性 | 高 | 中 |
| 采购欺诈 | 伪造采购订单、供应商信息进行欺诈 | 高 | 中 |
| 移动端数据泄露 | 移动端设备丢失或被盗导致库存数据泄露 | 中 | 高 |
| 扫码攻击 | 伪造备件二维码进行恶意库存操作 | 中 | 中 |
| 供应链攻击 | 通过供应商系统入侵备件管理系统 | 中 | 低 |
| 内部威胁 | 内部人员滥用权限修改库存数据 | 高 | 中 |

#### 1.2.2 攻击面分析
1. **Web API**: RESTful API端点、WebSocket实时库存通知
2. **移动端应用**: 微信小程序、H5、App客户端
3. **数据库**: 备件档案、库存数据、采购订单存储
4. **文件存储**: 备件图片、采购合同、质检报告
5. **二维码生成和解析**: 备件扫码功能
6. **打印服务**: 标签打印、报表打印
7. **供应商接口**: 供应商数据交换接口
8. **财务接口**: 成本数据同步接口

### 1.3 安全设计原则
1. **最小权限原则**: 用户仅拥有完成工作所需的最小权限
2. **职责分离**: 采购、入库、出库、盘点由不同角色负责
3. **库存完整性**: 库存操作事务保障，防止数据不一致
4. **成本数据保护**: 采购价格、成本数据严格权限控制
5. **操作审计**: 所有库存操作记录详细审计日志
6. **输入验证**: 所有用户输入进行严格验证和过滤

```python
class InventoryAccessPolicy:
    def can_view_inventory(self, user, inventory_item):
        """判断用户是否可以查看库存"""
        
        # 规则1: 管理员可以查看所有库存
        if user.has_role('admin'):
            return True
            
        # 规则2: 检查仓库权限
        if inventory_item.warehouse_id not in user.warehouse_scope:
            return False
            
        # 规则3: 检查成本数据权限
        if self.is_cost_data(inventory_item) and user.cost_access_level == 'none':
            # 脱敏处理
            return self.mask_cost_data(inventory_item)
            
        # 规则4: 检查场站权限
        warehouse = self.get_warehouse(inventory_item.warehouse_id)
        if warehouse.station_id and warehouse.station_id not in user.station_scope:
            return False
            
        return True
    
    def can_modify_inventory(self, user, inventory_item, operation_type):
        """判断用户是否可以修改库存"""
        
        # 库存锁定检查
        if inventory_item.is_locked:
            return False, "库存已被锁定"
            
        # 操作时间限制
        if operation_type == 'adjust' and not self.within_adjustment_window():
            return False, "不在库存调整时间窗口内"
            
        # 数量限制检查
        if operation_type == 'issue' and inventory_item.available_stock <= 0:
            return False, "库存不足"
            
        # 批次效期检查
        if operation_type == 'issue' and inventory_item.expiry_date and inventory_item.expiry_date < date.today():
            return False, "备件已过期"
            
        # 权限检查
        required_permission = self.get_required_permission(operation_type)
        if not user.has_permission(required_permission):
            return False, "权限不足"
            
        return True, None
    
    def mask_cost_data(self, inventory_item):
        """脱敏成本数据"""
        masked = inventory_item.copy()
        masked.unit_price = None
        masked.total_value = None
        masked.last_purchase_price = None
        return masked
```

3. **成本数据隔离**: 非财务人员查看成本数据脱敏

### 3.3 数据脱敏
```typescript
// 数据脱敏处理器
class InventoryDataMasking {
  static maskInventoryData(inventoryData: any, user: User): any {
    const masked = { ...inventoryData };
    
    // 成本数据脱敏
    if (user.cost_access_level !== 'full') {
      delete masked.unitPrice;
      delete masked.totalValue;
      delete masked.lastPurchasePrice;
      delete masked.standardCost;
      delete masked.avgPurchasePrice;
      
      // 采购订单脱敏
      if (masked.purchaseOrders) {
        masked.purchaseOrders = masked.purchaseOrders.map((order: any) => ({
          ...order,
          totalAmount: '***',
          items: order.items.map((item: any) => ({
            ...item,
            unitPrice: '***',
            lineTotal: '***'
          }))
        }));
      }
    }
    
    // 供应商信息脱敏
    if (!user.has_permission('supplier:view_details')) {
      if (masked.suppliers) {
        masked.suppliers = masked.suppliers.map((supplier: any) => ({
          ...supplier,
          contactPerson: this.maskName(supplier.contactPerson),
          phone: this.maskPhone(supplier.phone),
          email: this.maskEmail(supplier.email),
          bankAccount: '***'
        }));
      }
    }
    
    return masked;
  }
  
  static maskName(name: string): string {
    if (!name || name.length <= 2) return '**';
    return name.charAt(0) + '**' + name.charAt(name.length - 1);
  }
  
  static maskPhone(phone: string): string {
    if (!phone) return '***';
    return phone.replace(/(\d{3})\d{4}(\d{4})/, '$1****$2');
  }
}
```

```python
class InventoryFileUploadSecurity:
    ALLOWED_EXTENSIONS = {
        'image': ['.jpg', '.jpeg', '.png', '.gif'],
        'document': ['.pdf', '.doc', '.docx', '.xls', '.xlsx'],
        'other': ['.txt', '.csv']
    }
    MAX_FILE_SIZE = 20 * 1024 * 1024  # 20MB
    SCAN_FOR_MALWARE = True
    VALIDATE_CONTENT = True
    
    def validate_upload(self, file, upload_type):
        """验证文件上传"""
        
        # 1. 文件类型检查
        ext = os.path.splitext(file.filename)[1].lower()
        allowed_exts = self.ALLOWED_EXTENSIONS.get(upload_type, [])
        if ext not in allowed_exts:
            raise ValidationError(f'不支持的文件类型: {ext}')
        
        # 2. 文件大小检查
        if file.size > self.MAX_FILE_SIZE:
            raise ValidationError('文件大小超过限制')
        
        # 3. 文件名安全检查
        safe_filename = self.sanitize_filename(file.filename)
        
        # 4. 文件内容检查
        if self.SCAN_FOR_MALWARE:
            scan_result = self.scan_for_malware(file)
            if not scan_result.clean:
                raise ValidationError(f'文件包含恶意内容: {scan_result.threats}')
        
        # 5. 内容验证（针对特定类型）
        if self.VALIDATE_CONTENT:
            if upload_type == 'image':
                self.validate_image(file)
            elif upload_type == 'document':
                self.validate_document(file)
        
        # 6. 添加数字水印（图片）
        if upload_type == 'image':
            file_content = self.add_watermark(file)
        else:
            file_content = file.read()
        
        # 7. 加密存储
        encrypted_file = self.encrypt_file(file_content, safe_filename)
        
        return {
            'filename': safe_filename,
            'encrypted_content': encrypted_file,
            'file_hash': self.calculate_hash(file_content),
            'file_size': file.size,
            'content_type': file.content_type
        }
    
    def sanitize_filename(self, filename):
        """文件名安全处理"""
        # 移除路径信息
        basename = os.path.basename(filename)
        # 移除特殊字符
        safe_name = re.sub(r'[^\w\-\.]', '_', basename)
        # 限制长度
        safe_name = safe_name[:100]
        # 添加随机前缀
        random_prefix = secrets.token_hex(8)
        return f"{random_prefix}_{safe_name}"
```

### 4.3 会话安全

#### 4.3.1 会话管理

```typescript
class InventorySessionManager {
  private config = {
    cookieName: '__Secure-inventory-session',
    httpOnly: true,
    secure: true,
    sameSite: 'strict',
    maxAge: 15 * 60 * 1000,  // 15分钟
    rolling: true,
    domain: '.example.com',
    path: '/',
  };
  
  // 库存操作会话加强
  enforceInventorySession(operationType: string, session: Session): void {
    // 高价值操作需要重新认证
    const highValueOperations = [
      'cost_adjustment',
      'bulk_transfer',
      'major_adjustment',
      'supplier_payment'
    ];
    
    if (highValueOperations.includes(operationType)) {
      // 检查会话新鲜度
      const sessionAge = Date.now() - session.createdAt;
      if (sessionAge > 5 * 60 * 1000) {  // 5分钟
        throw new ReauthenticationRequiredError('需要重新认证');
      }
      
      // 检查操作频率
      const recentOperations = this.getRecentOperations(session.userId, operationType);
      if (recentOperations.length > 3) {  // 5分钟内超过3次
        throw new RateLimitError('操作过于频繁');
      }
    }
    
    // 记录操作会话
    this.recordOperationSession(session.id, operationType);
  }
  
  // 并发会话控制
```

```python
class InventoryConcurrencyControl:
    def execute_inventory_operation(self, operation: InventoryOperation):
        """执行库存操作（带并发控制）"""
        
        # 获取库存锁
        lock_key = f"inventory_lock:{operation.spare_part_id}:{operation.warehouse_id}"
        lock_acquired = self.redis.setnx(lock_key, operation.id)
        
        if not lock_acquired:
            # 检查是否死锁
            lock_owner = self.redis.get(lock_key)
            lock_age = self.redis.ttl(lock_key)
            
            if lock_age < 0:  # 锁已过期
                self.redis.delete(lock_key)
                lock_acquired = self.redis.setnx(lock_key, operation.id)
            else:
                raise ConcurrentModificationError(
                    f"库存正在被操作{lock_owner}修改，请稍后重试"
                )
        
        try:
            # 设置锁过期时间（30秒）
            self.redis.expire(lock_key, 30)
            
            # 使用数据库事务
            with self.db.transaction():
                # 检查库存
                current_stock = self.get_current_stock(
                    operation.spare_part_id,
                    operation.warehouse_id
                )
                
                # 验证操作
                if operation.operation_type in ['issue', 'transfer_out']:
                    if current_stock.available_stock < operation.quantity:
                        raise InsufficientStockError(
                            f"库存不足，可用库存: {current_stock.available_stock}"
                        )
                
                # 执行操作
                result = self.execute_operation(operation)
                
                # 记录审计日志
                self.audit_logger.log_inventory_operation(operation, result)
                
                return result
                
        finally:
            # 释放锁
            if lock_acquired:
                self.redis.delete(lock_key)
    
    def optimistic_concurrency_control(self, inventory_item, update_fn):
        """乐观并发控制"""
        
        version = inventory_item.version
        
        # 执行更新
        updated = update_fn(inventory_item)
        updated.version = version + 1
        
        # 使用版本号检查并发
        result = self.db.execute(
            "UPDATE inventory_snapshots SET version = :new_version, ... "
            "WHERE id = :id AND version = :old_version",
            {
                'id': inventory_item.id,
                'old_version': version,
                'new_version': version + 1
            }
        )
        
        if result.rowcount == 0:
            raise StaleDataError("数据已被其他用户修改，请刷新后重试")
```

```python
            # 出库验证
            if not operation.work_order_id:

                batch_info = self.get_batch_info(operation.batch_no)

        if not self.has_operation_permission(operation):

        if not self.within_operation_window(operation):
            errors.append("不在允许的操作时间窗口内")
        
        # 5. 库存验证（实时检查）
```

   - 单次库存调整金额超过阈值（如5万元）
   - 非工作时间的大额操作
   - 高频库存调整操作

2. **数据告警**：
   - 库存数据不一致告警
   - 成本数据异常变动
   - 供应商风险评分突变
   - 采购价格异常波动

3. **系统告警**：
   - 库存超卖风险
   - 并发操作冲突频繁
   - 审计日志写入失败
   - 安全策略执行失败

## 8. 应急响应

### 8.1 安全事件分类
```yaml
安全事件等级:
  一级 (严重):
    - 系统被入侵，攻击者获取管理员权限
    - 大量库存数据被篡改或删除
    - 采购欺诈造成重大经济损失
    - 系统完全不可用
    
  二级 (高危):
    - 用户账户被盗用进行恶意操作
    - 敏感成本数据泄露
    - 库存数据不一致影响业务
    - 部分功能不可用
    
  三级 (中危):
    - 暴力破解攻击尝试
    - 扫描攻击探测
    - 轻微数据泄露
    - 性能问题影响使用
    
  四级 (低危):
    - 配置错误导致的信息泄露
    - 轻微权限问题
    - 操作错误记录
```

### 8.2 应急响应流程
```
安全事件发现
       ↓
  初步评估和定级
       ↓
  启动应急响应计划
       ↓
  遏制和隔离影响
       ↓
  取证和调查分析
       ↓
  恢复和修复系统
       ↓
  事后总结和改进
```

### 8.3 数据备份和恢复
1. **备份策略**：
   - 实时备份：数据库变更实时同步到备份库
   - 每日全量备份：凌晨进行全量备份
   - 交易日志备份：所有库存操作日志备份
   - 异地备份：备份数据同步到异地机房

2. **恢复测试**：
   - 每月进行一次恢复演练
   - 验证备份数据的完整性和可用性
   - 测试不同恢复点目标（RPO）和恢复时间目标（RTO）
   - 记录恢复时间和成功率

## 9. 安全测试

### 9.1 测试类型
1. **静态应用安全测试 (SAST)**：代码层面安全漏洞扫描
2. **动态应用安全测试 (DAST)**：运行时安全测试
3. **移动应用安全测试 (MAST)**：移动端应用安全测试
4. **渗透测试**：模拟攻击者进行安全测试
5. **库存业务逻辑测试**：库存操作业务逻辑安全测试

### 9.2 测试频率
- **每次提交**：SAST自动扫描
- **每周**：DAST自动扫描
- **每月**：人工安全代码审查
- **每季度**：全面渗透测试
- **每次发布**：安全回归测试
- **每年**：第三方安全审计

### 9.3 测试报告和修复
1. **漏洞管理**：所有发现漏洞进入漏洞管理系统
2. **优先级排序**：根据CVSS评分和业务影响确定修复优先级
3. **修复跟踪**：跟踪漏洞修复进度，设置修复时限
4. **验证测试**：修复后验证漏洞是否真正修复
5. **知识积累**：将测试经验转化为安全开发规范
