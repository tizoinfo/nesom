**技术栈**: Vue 3.4 + Python 3.11 + FastAPI + MySQL 8.0 + Redis + Docker

## 1. 设计概述

### 1.1 设计目标
备件管理模块是NESOM系统的物资管理核心，负责：
- 备件库存管理和预警机制
- 备件分类和编码体系管理
- 入库、出库、调拨、盘点流程设计
- 供应商管理和采购流程
- 备件生命周期管理（采购、入库、领用、报废）
- 成本核算和库存优化

### 1.2 设计原则
1. **库存准确性**：实时库存数据，事务保障数据一致性
2. **流程完整性**：覆盖备件从采购到报废的全生命周期
3. **成本精细化管理**：支持多种计价方法（先进先出、加权平均）
4. **移动端支持**：支持扫码出入库、移动盘点
5. **预警智能化**：库存预警、效期预警、呆滞料预警

### 1.3 与其他模块关系
- **依赖模块**：用户权限管理（库管员、采购员）、设备监控模块（设备信息）
- **关联模块**：工单管理模块（备件领用）、采购管理模块（采购流程）
- **数据提供**：报表统计模块（库存统计、成本分析）
- **集成模块**：财务系统（成本核算）、供应商系统（供应商信息）

## 2. 表结构详细设计

### 2.1 备件分类表 (spare_part_categories)
备件分类体系，支持树形结构，定义备件属性和管理策略。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID() | 主键 |
| category_code | VARCHAR(50) | UNIQUE NOT NULL | - | 分类编码，唯一标识 |
| category_name | VARCHAR(100) | NOT NULL | - | 分类名称 |
| parent_id | VARCHAR(36) | FOREIGN KEY | NULL | 父分类ID，支持树形结构 |
| level | INT | NOT NULL | 1 | 分类层级 |
| sort_order | INT | NOT NULL | 0 | 显示排序 |
| is_leaf | BOOLEAN | NOT NULL | FALSE | 是否叶子分类 |
| attribute_template | JSON | - | NULL | 属性模板，定义备件属性 |
| storage_requirements | JSON | - | NULL | 存储要求（温湿度、防静电等） |
| unit | VARCHAR(20) | NOT NULL | 'piece' | 默认计量单位 |
| min_stock_level | DECIMAL(10,3) | - | NULL | 最低库存量 |
| max_stock_level | DECIMAL(10,3) | - | NULL | 最高库存量 |
| safety_stock_level | DECIMAL(10,3) | - | NULL | 安全库存量 |
| lead_time_days | INT | - | NULL | 采购提前期（天） |
| abc_classification | ENUM('A','B','C') | INDEX | 'C' | ABC分类（价值分类） |
| lifecycle_stage | ENUM('introduction','growth','maturity','decline','obsolete') | - | 'maturity' | 生命周期阶段 |
| description | TEXT | - | NULL | 分类描述 |
| icon | VARCHAR(255) | - | NULL | 分类图标 |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**外键约束**：
- `FOREIGN KEY (parent_id) REFERENCES spare_part_categories(id) ON DELETE CASCADE`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 唯一索引：`UNIQUE KEY uk_category_code (category_code)`
- 查询索引：`INDEX idx_parent_id (parent_id)`, `INDEX idx_abc_classification (abc_classification)`
- 层级索引：`INDEX idx_level (level)` 用于层级查询

**业务规则**：
1. 分类编码全局唯一，格式：`CAT-{大类}-{中类}-{小类}`
2. 树形结构最大深度不超过5层
3. 叶子分类才能关联备件
4. attribute_template定义备件动态属性验证规则

- `FOREIGN KEY (category_id) REFERENCES spare_part_categories(id) ON DELETE RESTRICT`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 唯一索引：`UNIQUE KEY uk_spare_part_code (spare_part_code)`
- 查询索引：`INDEX idx_category (category_id)`, `INDEX idx_brand (brand)`, `INDEX idx_model (model)`
- 状态索引：`INDEX idx_status (status)`
- 全文索引：`FULLTEXT idx_search (spare_part_name, specification, model)` 用于搜索

**业务规则**：
1. 备件编码全局唯一，格式：`SP-{分类码}-{序列号}`
2. 当前库存 = 所有仓库库存之和，实时计算
3. 可用库存 = 当前库存 - 预留库存
4. 受控件（is_controlled=true）需要序列号追踪
5. 消耗品（is_consumable=true）按数量管理，非消耗品按件管理

| level_code | VARCHAR(10) | NOT NULL | - | 层号 |

1. 库位编码格式：`{区域}-{排}-{架}-{层}-{位}`，如：`A-01-02-03-04`
2. 库位状态流转：available → reserved → occupied → available
3. 库位容量支持按重量、体积、数量多种维度控制
4. 危险品必须存储在相应危险等级的库位

| operator_id | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 操作人ID |
| operator_name | VARCHAR(100) | NOT NULL | - | 操作人姓名 |
| reference_no | VARCHAR(100) | INDEX | NULL | 参考单号 |
| remarks | TEXT | - | NULL | 备注 |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |

**外键约束**：
- `FOREIGN KEY (spare_part_id) REFERENCES spare_parts(id) ON DELETE RESTRICT`
- `FOREIGN KEY (warehouse_id) REFERENCES warehouses(id) ON DELETE RESTRICT`
- `FOREIGN KEY (location_id) REFERENCES storage_locations(id) ON DELETE SET NULL`
- `FOREIGN KEY (supplier_id) REFERENCES suppliers(id) ON DELETE SET NULL`
- `FOREIGN KEY (work_order_id) REFERENCES work_orders(id) ON DELETE SET NULL`
- `FOREIGN KEY (transfer_order_id) REFERENCES transfer_orders(id) ON DELETE SET NULL`
- `FOREIGN KEY (inventory_order_id) REFERENCES inventory_orders(id) ON DELETE SET NULL`
- `FOREIGN KEY (operator_id) REFERENCES users(id) ON DELETE RESTRICT`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 唯一索引：`UNIQUE KEY uk_transaction_no (transaction_no)`
- 查询索引：`INDEX idx_spare_part (spare_part_id)`, `INDEX idx_warehouse (warehouse_id)`, `INDEX idx_transaction_date (transaction_date DESC)`
- 组合索引：`INDEX idx_spare_warehouse (spare_part_id, warehouse_id)` 用于查询库存
- 批次索引：`INDEX idx_batch (batch_no)` 用于批次追踪

**业务规则**：
1. 事务编号全局唯一，格式：`TR-{年月日}-{类型}-{序列号}`
2. 数量正数表示入库，负数表示出库
3. 受控件（has_serial_number=true）必须记录序列号
4. 有效期管理：记录生产日期和有效期，支持近效期预警
5. 支持先进先出（FIFO）、批次管理等库存策略

- 查询索引：`INDEX idx_spare_warehouse (spare_part_id, warehouse_id)`, `INDEX idx_status (status)`

- 序列号索引：`INDEX idx_serial (serial_number)` 用于序列号追踪

**业务规则**：
1. 库存快照实时更新，每次库存变动后重新计算
2. 支持批次、库位、序列号维度的精细库存管理
3. 库存状态：可用、预留、隔离（质检）、冻结
4. 乐观锁机制防止并发更新冲突

## 6. 索引和查询优化策略

### 6.1 高性能查询场景
1. **库存查询**：`WHERE spare_part_id=? AND warehouse_id=?` → 索引: (spare_part_id, warehouse_id)
2. **备件搜索**：`WHERE spare_part_name LIKE ? OR specification LIKE ?` → 全文索引
3. **库存预警**：`WHERE available_stock < min_stock_level` → 索引: (available_stock)
4. **效期预警**：`WHERE expiry_date BETWEEN ? AND ?` → 索引: (expiry_date)

### 6.2 分区策略
1. **库存明细表**：按月分区，保留36个月数据
2. **库存快照表**：按仓库ID哈希分区
3. **历史数据归档**：3年前数据移至历史表

### 6.3 查询优化建议
1. **避免全表扫描**：WHERE条件使用索引字段
2. **分页优化**：使用基于主键的游标分页
3. **聚合查询**：定期预计算库存统计指标
4. **读写分离**：报表查询走只读副本
