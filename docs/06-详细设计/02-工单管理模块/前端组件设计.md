# 工单管理模块 - 前端组件详细设计

**版本**: 1.0  
**日期**: 2026-04-02  
**作者**: 高级项目经理  
**状态**: Draft  
**审核状态**: 待评审  
**技术栈**: Vue 3.4 + TypeScript 5.3 + Element Plus 2.7 + Vite  
**设计系统**: NESOM Design System (基于Element Plus扩展)  
**响应式支持**: Desktop (≥1024px), Tablet (768px-1023px), Mobile (≤767px)  
**移动端框架**: Uni-app (支持微信小程序、H5、App)

## 1. 设计概述

### 1.1 设计目标
工单管理前端模块提供高效、直观、易用的工单管理界面，支持：
- **工单全生命周期管理**：创建、分配、处理、审核、关闭全流程
- **多端适配**：PC端管理后台 + 移动端现场作业
- **实时协作**：工单状态实时同步，消息及时通知
- **数据可视化**：工单统计、绩效分析可视化展示
- **移动端优化**：扫码、拍照、定位、离线操作

### 1.2 设计原则
1. **流程导向**：界面设计跟随工单业务流程
2. **角色适配**：不同角色（调度员、技工、审核人）界面差异化
3. **移动优先**：关键操作移动端友好，支持离线
4. **实时反馈**：工单状态变更实时提示
5. **操作高效**：常用操作快捷入口，减少点击
6. **数据安全**：敏感操作二次确认，操作记录可追溯

### 1.3 技术约束
- **PC端框架**: Vue 3.4 Composition API + `<script setup>`
- **移动端框架**: Uni-app + Vue 3 (一套代码多端发布)
- **语言**: TypeScript 5.3 (严格模式)
- **UI库**: Element Plus 2.7 (PC) + uView (移动端)
- **状态管理**: Pinia (共享状态管理)
- **路由**: Vue Router 4 (PC) + Uni-app路由 (移动端)
- **HTTP客户端**: Axios + 自定义拦截器
- **WebSocket**: Socket.io (实时通知)
- **图表**: ECharts 5 (PC) + uCharts (移动端)
- **地图**: 百度地图API (PC) + 微信小程序地图 (移动端)
- **扫码**: HTML5 QR Code Scanner (PC) + 原生扫码 (移动端)

## 3. 核心组件设计

### 3.1 工单列表组件 (WorkOrderList)

#### 3.1.1 组件概述
- **用途**: 工单列表展示、筛选、排序、批量操作
- **使用场景**: PC端工单管理、移动端我的工单
- **变体**: 卡片列表、表格列表、看板视图

#### 3.1.2 属性定义
```typescript
interface WorkOrderListProps {
  // 数据源
  workOrders: WorkOrderItem[];  // 工单数据
  loading: boolean;             // 加载状态
  
  // 配置
  viewMode: 'table' | 'card' | 'kanban';  // 视图模式
  showFilters: boolean;         // 是否显示筛选器
  showActions: boolean;         // 是否显示操作按钮
  selectable: boolean;          // 是否可多选
  
  // 筛选条件
  filters: WorkOrderFilters;    // 筛选条件
  sortBy: string;               // 排序字段
  sortOrder: 'asc' | 'desc';    // 排序方向
  
  // 分页
  pagination: Pagination;       // 分页信息
}

interface WorkOrderItem {
  id: string;
  workOrderNo: string;
  title: string;
  workOrderType: WorkOrderType;
  status: WorkOrderStatus;
  priority: WorkOrderPriority;
  emergencyLevel: EmergencyLevel;
  deviceName?: string;
  deviceCode?: string;
  reportedByName: string;
  reportedAt: string;
  assignedToName?: string;
  scheduledStart?: string;
  scheduledEnd?: string;
  completionRate: number;
  location?: string;
  overdue: boolean;             // 是否超时
}
```

#### 3.1.3 功能特性
1. **多视图模式**：
   - 表格视图：适合PC端管理，支持多列排序
   - 卡片视图：适合移动端，信息直观
   - 看板视图：按状态分组，拖拽状态变更

2. **智能筛选**：
   - 快速筛选：状态、类型、优先级
   - 高级筛选：时间范围、设备、执行人
   - 保存筛选：常用筛选条件保存

3. **批量操作**：
   - 批量分配：选择多个工单分配给人
   - 批量导出：导出选中工单数据
   - 批量状态变更：批量提交、取消

4. **实时更新**：
   - WebSocket监听工单状态变更
   - 自动刷新列表数据
   - 新工单提醒

## 4. 移动端专用组件

### 4.1 移动端扫码组件 (MobileQRScanner)

#### 4.1.1 组件概述
- **用途**: 移动端扫码识别工单、设备、备件
- **使用场景**: 工单扫码开始、备件扫码领用、设备扫码查看
- **平台支持**: 微信小程序、H5、App

#### 4.1.2 功能特性
1. **多码识别**：支持QR码、条形码、自定义码
2. **光线适应**：自动调节摄像头参数
3. **结果处理**：扫码后自动跳转对应页面
4. **历史记录**：保存扫码记录
5. **离线扫码**：离线扫码结果暂存，联网后同步

### 4.2 移动端拍照组件 (MobileCamera)

#### 4.2.1 组件概述
- **用途**: 移动端拍照上传，支持现场照片记录
- **使用场景**: 故障拍照、过程记录、完工确认
- **平台支持**: 微信小程序、H5、App

#### 4.2.2 功能特性
1. **照片分类**：预设照片分类（故障、过程、结果）
2. **照片处理**：自动压缩、添加水印、记录位置
3. **批量上传**：支持多张照片批量上传
4. **离线保存**：网络不佳时本地保存，自动重试上传

### 4.3 移动端离线组件 (MobileOfflineManager)

#### 4.3.1 组件概述
- **用途**: 移动端离线数据管理和同步
- **使用场景**: 无网络环境下的工单操作
- **平台支持**: 微信小程序、H5、App

#### 4.3.2 功能特性
1. **数据缓存**：缓存常用工单数据
2. **离线操作**：支持离线添加工单步骤
3. **冲突解决**：网络恢复后自动同步，冲突智能解决
4. **存储管理**：自动清理过期缓存数据
