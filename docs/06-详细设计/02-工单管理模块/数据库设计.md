## 1. 设计概述

### 1.1 设计目标
工单管理模块是NESOM系统的业务核心，负责：
- 工单全生命周期管理（创建、分配、处理、关闭、归档）
- 支持多种工单类型（维修、巡检、保养、故障）
- 工单优先级和紧急程度管理
- 工单流转状态机设计
- 工单与设备、备件、人员的关联关系
- 工单统计和绩效分析

### 1.2 设计原则
1. **业务流程完整**：覆盖工单从创建到归档的全流程
2. **数据一致性**：事务保障工单与关联数据的一致性
3. **性能优化**：高频操作优化，合理的索引设计
4. **可追溯性**：完整的操作日志和状态变更记录
5. **移动端支持**：支持扫码、拍照、位置等移动端特性

### 1.3 与其他模块关系
- **依赖模块**：用户权限管理（执行人员）、设备监控模块（设备信息）
- **关联模块**：备件管理模块（备件领用）、巡检管理模块（巡检计划）
- **数据提供**：报表统计模块（工单统计、绩效分析）
- **集成模块**：消息通知（工单状态变更通知）

## 2. 表结构详细设计

### 2.1 工单主表 (work_orders)
工单核心信息表，存储工单基本属性和生命周期状态。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID() | 主键 |
| work_order_no | VARCHAR(50) | UNIQUE NOT NULL | - | 工单编号，唯一标识 |
| work_order_type | ENUM('repair','inspection','maintenance','fault','other') | NOT NULL | 'repair' | 工单类型 |
| title | VARCHAR(200) | NOT NULL | - | 工单标题 |
| description | TEXT | NOT NULL | - | 工单描述 |
| status | ENUM('draft','pending','assigned','in_progress','pending_review','completed','closed','cancelled') | NOT NULL INDEX | 'draft' | 工单状态 |
| priority | ENUM('low','medium','high','emergency') | NOT NULL INDEX | 'medium' | 优先级 |
| emergency_level | ENUM('normal','urgent','critical') | NOT NULL | 'normal' | 紧急程度 |
| station_id | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 所属场站ID |
| device_id | VARCHAR(36) | FOREIGN KEY | NULL | 关联设备ID |
| device_name | VARCHAR(100) | - | NULL | 设备名称（冗余，方便查询） |
| device_code | VARCHAR(50) | - | NULL | 设备编码（冗余） |
| reported_by | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 上报人ID |
| reported_by_name | VARCHAR(100) | NOT NULL | - | 上报人姓名 |
| reported_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 上报时间 |
| assigned_to | VARCHAR(36) | FOREIGN KEY | NULL | 分配给（执行人ID） |
| assigned_to_name | VARCHAR(100) | - | NULL | 执行人姓名 |
| assigned_at | DATETIME | - | NULL | 分配时间 |
| scheduled_start | DATETIME | - | NULL | 计划开始时间 |
| scheduled_end | DATETIME | - | NULL | 计划完成时间 |
| actual_start | DATETIME | - | NULL | 实际开始时间 |
| actual_end | DATETIME | - | NULL | 实际完成时间 |
| estimated_duration | INT | - | NULL | 预计耗时（分钟） |
| actual_duration | INT | - | NULL | 实际耗时（分钟） |
| completion_rate | INT | - | 0 | 完成进度（0-100） |
| cost_estimate | DECIMAL(12,2) | - | NULL | 预估成本 |
| actual_cost | DECIMAL(12,2) | - | NULL | 实际成本 |
| location | VARCHAR(255) | - | NULL | 工单位置 |
| longitude | DECIMAL(10,6) | - | NULL | 经度（移动端定位） |
| latitude | DECIMAL(10,6) | - | NULL | 纬度（移动端定位） |
| qr_code | VARCHAR(255) | - | NULL | 二维码内容 |
| images | JSON | - | NULL | 工单相关图片 |
| attachments | JSON | - | NULL | 附件列表 |
| tags | JSON | - | NULL | 标签（扩展属性） |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| closed_at | DATETIME | - | NULL | 关闭时间 |
| closed_by | VARCHAR(36) | - | NULL | 关闭人ID |
| archived_at | DATETIME | - | NULL | 归档时间 |

**外键约束**：
- `FOREIGN KEY (station_id) REFERENCES stations(id) ON DELETE RESTRICT`
- `FOREIGN KEY (device_id) REFERENCES devices(id) ON DELETE SET NULL`
- `FOREIGN KEY (reported_by) REFERENCES users(id) ON DELETE RESTRICT`
- `FOREIGN KEY (assigned_to) REFERENCES users(id) ON DELETE SET NULL`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 唯一索引：`UNIQUE KEY uk_work_order_no (work_order_no)`
- 查询索引：`INDEX idx_status (status)`, `INDEX idx_priority (priority)`, `INDEX idx_type (work_order_type)`
- 组合索引：`INDEX idx_station_status (station_id, status)`, `INDEX idx_assigned_status (assigned_to, status)`
- 时间索引：`INDEX idx_reported_at (reported_at DESC)`, `INDEX idx_scheduled_start (scheduled_start)`

**业务规则**：
1. 工单编号格式：`WO-{场站代码}-{年月日}-{序列号}`，唯一且不可变
2. 状态流转必须符合状态机定义（见业务逻辑设计）
3. 紧急程度影响工单处理时限：normal(24h)、urgent(8h)、critical(2h)
4. 实际开始时间由执行人确认开始工作时填写
5. 实际结束时间由执行人提交完成时填写
6. 工单关闭后进入只读状态，不能再修改

### 2.2 工单明细表 (work_order_details)
工单处理过程详细记录，支持多次处理记录。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | BIGINT | PRIMARY KEY AUTO_INCREMENT | - | 自增主键 |
| work_order_id | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 工单ID |
| step_number | INT | NOT NULL | 1 | 步骤序号 |
| step_title | VARCHAR(100) | NOT NULL | - | 步骤标题 |
| step_description | TEXT | - | NULL | 步骤描述 |
| performed_by | VARCHAR(36) | FOREIGN KEY | NULL | 执行人ID |
| performed_by_name | VARCHAR(100) | - | NULL | 执行人姓名 |
| started_at | DATETIME | - | NULL | 开始时间 |
| completed_at | DATETIME | - | NULL | 完成时间 |
| duration_minutes | INT | - | NULL | 耗时（分钟） |
| findings | TEXT | - | NULL | 检查发现 |
| actions_taken | TEXT | - | NULL | 采取的措施 |
| tools_used | JSON | - | NULL | 使用工具列表 |
| parts_used | JSON | - | NULL | 使用备件列表（关联备件管理） |
| before_status | VARCHAR(50) | - | NULL | 处理前状态 |
| after_status | VARCHAR(50) | - | NULL | 处理后状态 |
| before_images | JSON | - | NULL | 处理前照片 |
| after_images | JSON | - | NULL | 处理后照片 |
| test_results | JSON | - | NULL | 测试结果 |
| notes | TEXT | - | NULL | 备注 |
| quality_check | ENUM('pass','fail','pending') | - | 'pending' | 质量检查结果 |
| quality_check_by | VARCHAR(36) | - | NULL | 质量检查人 |
| quality_check_at | DATETIME | - | NULL | 检查时间 |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**外键约束**：
- `FOREIGN KEY (work_order_id) REFERENCES work_orders(id) ON DELETE CASCADE`
- `FOREIGN KEY (performed_by) REFERENCES users(id) ON DELETE SET NULL`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 查询索引：`INDEX idx_work_order (work_order_id)`, `INDEX idx_performed_by (performed_by)`
- 排序索引：`INDEX idx_work_order_step (work_order_id, step_number)`

**业务规则**：
1. 一个工单可以有多个处理步骤（如：诊断、维修、测试）
2. 每个步骤可以关联多个备件（parts_used存储备件ID和数量）
3. 质量检查通过后工单才能进入completed状态
4. 步骤可以并行或串行执行，由step_number控制顺序

### 2.3 工单状态历史表 (work_order_status_history)
工单状态变更历史记录，用于追踪和审计。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | BIGINT | PRIMARY KEY AUTO_INCREMENT | - | 自增主键 |
| work_order_id | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 工单ID |
| old_status | VARCHAR(50) | NOT NULL | - | 原状态 |
| new_status | VARCHAR(50) | NOT NULL | - | 新状态 |
| changed_by | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 变更人ID |
| changed_by_name | VARCHAR(100) | NOT NULL | - | 变更人姓名 |
| change_reason | VARCHAR(255) | - | NULL | 变更原因 |
| change_notes | TEXT | - | NULL | 变更备注 |
| changed_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 变更时间 |
| metadata | JSON | - | NULL | 变更时的附加数据 |

**外键约束**：
- `FOREIGN KEY (work_order_id) REFERENCES work_orders(id) ON DELETE CASCADE`
- `FOREIGN KEY (changed_by) REFERENCES users(id) ON DELETE RESTRICT`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 查询索引：`INDEX idx_work_order (work_order_id)`, `INDEX idx_changed_at (changed_at DESC)`
- 状态索引：`INDEX idx_status_change (old_status, new_status)`

**业务规则**：
1. 每次工单状态变更必须记录
2. 变更原因对于特定状态变更为必填项（如：取消、驳回）
3. 提供完整的审计追溯能力
4. 支持状态变更的统计和分析

### 2.4 工单评价表 (work_order_evaluations)
工单完成后的客户评价和服务质量反馈。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | BIGINT | PRIMARY KEY AUTO_INCREMENT | - | 自增主键 |
| work_order_id | VARCHAR(36) | FOREIGN KEY UNIQUE NOT NULL | - | 工单ID |
| evaluator_id | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 评价人ID |
| evaluator_name | VARCHAR(100) | NOT NULL | - | 评价人姓名 |
| evaluator_role | ENUM('customer','supervisor','colleague','system') | NOT NULL | 'customer' | 评价人角色 |
| satisfaction_score | INT | NOT NULL | - | 满意度评分（1-5分） |
| timeliness_score | INT | NOT NULL | - | 及时性评分（1-5分） |
| quality_score | INT | NOT NULL | - | 质量评分（1-5分） |
| communication_score | INT | NOT NULL | - | 沟通评分（1-5分） |
| overall_score | DECIMAL(3,1) | NOT NULL | - | 综合评分 |
| positive_comments | TEXT | - | NULL | 正面评价 |
| improvement_suggestions | TEXT | - | NULL | 改进建议 |
| would_recommend | BOOLEAN | - | NULL | 是否推荐 |
| evaluated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 评价时间 |
| response_by | VARCHAR(36) | - | NULL | 回复人ID |
| response_content | TEXT | - | NULL | 回复内容 |
| response_at | DATETIME | - | NULL | 回复时间 |

**外键约束**：
- `FOREIGN KEY (work_order_id) REFERENCES work_orders(id) ON DELETE CASCADE`
- `FOREIGN KEY (evaluator_id) REFERENCES users(id) ON DELETE RESTRICT`
- `FOREIGN KEY (response_by) REFERENCES users(id) ON DELETE SET NULL`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 唯一索引：`UNIQUE KEY uk_work_order (work_order_id)` 一个工单只能评价一次
- 查询索引：`INDEX idx_scores (overall_score DESC)`, `INDEX idx_evaluated_at (evaluated_at DESC)`

**业务规则**：
1. 工单完成并关闭后才可以评价
2. 评价人可以是客户、主管或同事
3. 综合评分自动计算：`(satisfaction+timeliness+quality+communication)/4`
4. 评价后可以回复，形成双向沟通

## 4. 数据字典

### 4.1 工单状态枚举 (work_orders.status)
| 值 | 描述 | 业务含义 | 允许的下一个状态 |
|----|------|----------|-----------------|
| draft | 草稿 | 工单创建中，未提交 | pending, cancelled |
| pending | 待分配 | 已提交，等待分配 | assigned, cancelled |
| assigned | 已分配 | 已分配执行人 | in_progress, cancelled |
| in_progress | 进行中 | 执行人已开始处理 | pending_review, cancelled |
| pending_review | 待审核 | 处理完成，等待审核 | completed, in_progress, cancelled |
| completed | 已完成 | 审核通过，工单完成 | closed, cancelled |
| closed | 已关闭 | 工单关闭，进入历史 | - |
| cancelled | 已取消 | 工单取消 | - |

### 4.2 工单类型枚举 (work_orders.work_order_type)
| 值 | 描述 | 适用场景 | 默认优先级 |
|----|------|----------|------------|
| repair | 维修 | 设备故障修复 | high |
| inspection | 巡检 | 定期设备检查 | medium |
| maintenance | 保养 | 预防性维护 | medium |
| fault | 故障 | 紧急故障处理 | emergency |
| other | 其他 | 其他类型工单 | low |

### 4.3 优先级枚举 (work_orders.priority)
| 值 | 描述 | 处理时限 | 通知方式 |
|----|------|----------|----------|
| low | 低 | 72小时 | 站内信 |
| medium | 中 | 24小时 | 站内信+邮件 |
| high | 高 | 8小时 | 站内信+邮件+短信 |
| emergency | 紧急 | 2小时 | 站内信+邮件+短信+电话 |

### 4.4 紧急程度枚举 (work_orders.emergency_level)

## 6. 索引和查询优化策略

### 6.1 高性能查询场景
1. **工单列表查询**：`WHERE station_id=? AND status IN (?)` → 索引: (station_id, status)
2. **我的工单查询**：`WHERE assigned_to=? AND status IN ('assigned','in_progress')` → 索引: (assigned_to, status)
3. **超时工单查询**：`WHERE status NOT IN ('closed','cancelled') AND scheduled_end < NOW()` → 索引: (status, scheduled_end)
4. **统计查询**：`WHERE reported_at BETWEEN ? AND ?` → 索引: (reported_at)

### 6.2 分区策略
1. **状态历史表**：按月分区，保留24个月数据
2. **工单明细表**：按工单创建年份分区
3. **归档策略**：closed状态超过6个月的工单移至归档表

### 6.3 查询优化建议
1. **避免全表扫描**：WHERE条件使用索引字段
2. **分页优化**：使用基于created_at的游标分页
3. **聚合查询**：定期预计算工单统计指标
4. **读写分离**：报表查询走只读副本
