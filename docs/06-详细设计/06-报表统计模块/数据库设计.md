## 1. 设计概述

### 1.1 设计目标
报表统计模块是NESOM系统的数据分析核心，负责：
- 运行数据统计和绩效分析
- 自定义报表生成和导出
- 数据可视化和仪表盘
- 成本效益分析和预测
- 多维度数据钻取和分析

### 1.2 设计原则
1. **性能优先**：大数据量聚合表设计，预计算统计表，多级缓存策略
2. **灵活性**：支持拖拽式报表配置，动态数据源集成
3. **可扩展性**：插件式图表类型，自定义计算指标
4. **安全性**：行级数据权限控制，查询审计追踪
5. **用户体验**：响应式仪表盘，实时数据刷新，个性化收藏

### 1.3 与其他模块关系
- **数据来源**：
  - 设备监控模块：运行数据、告警数据、状态数据
  - 工单管理模块：维修数据、成本数据、工时数据
  - 巡检管理模块：巡检结果、问题数据、合规数据
  - 备件管理模块：库存数据、采购数据、成本数据
- **数据提供**：所有模块的统计分析和可视化展示

## 2. 表结构详细设计

## 3. 性能优化设计

### 3.1 大数据量聚合表设计
为高频统计查询创建预聚合表：

**日度统计聚合表 (daily_statistics_agg)**
```sql
CREATE TABLE daily_statistics_agg (
    agg_date DATE NOT NULL,
    station_id VARCHAR(36) NOT NULL,
    device_type_id VARCHAR(36) NOT NULL,
    metric_type VARCHAR(50) NOT NULL,
    value_sum DECIMAL(20,4),
    value_avg DECIMAL(20,4),
    value_min DECIMAL(20,4),
    value_max DECIMAL(20,4),
    record_count INT,
    PRIMARY KEY (agg_date, station_id, device_type_id, metric_type),
    INDEX idx_agg_date (agg_date),
    INDEX idx_station_metric (station_id, metric_type)
) ENGINE=InnoDB PARTITION BY RANGE (YEAR(agg_date)) (
    PARTITION p2026 VALUES LESS THAN (2027),
    PARTITION p2027 VALUES LESS THAN (2028)
);
```

**月度统计聚合表 (monthly_statistics_agg)**
```sql
CREATE TABLE monthly_statistics_agg (
    agg_year INT NOT NULL,
    agg_month INT NOT NULL,
    station_id VARCHAR(36) NOT NULL,
    device_type_id VARCHAR(36) NOT NULL,
    metric_type VARCHAR(50) NOT NULL,
    value_sum DECIMAL(20,4),
    value_avg DECIMAL(20,4),
    value_min DECIMAL(20,4),
    value_max DECIMAL(20,4),
    record_count INT,
    PRIMARY KEY (agg_year, agg_month, station_id, device_type_id, metric_type)
) ENGINE=InnoDB;
```

### 3.2 预计算统计表
定时任务预计算热点数据：

**设备运行效率预计算表 (device_efficiency_precalc)**
```sql
CREATE TABLE device_efficiency_precalc (
    calc_date DATE NOT NULL,
    device_id VARCHAR(36) NOT NULL,
    availability_rate DECIMAL(5,2) COMMENT '可用率(%)',
    performance_rate DECIMAL(5,2) COMMENT '性能率(%)',
    quality_rate DECIMAL(5,2) COMMENT '质量率(%)',
    oee DECIMAL(5,2) COMMENT '整体设备效率(%)',
    runtime_hours DECIMAL(8,2) COMMENT '运行时长(小时)',
    downtime_hours DECIMAL(8,2) COMMENT '停机时长(小时)',
    energy_output DECIMAL(12,2) COMMENT '能源产出(kWh)',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (calc_date, device_id),
    INDEX idx_device_date (device_id, calc_date)
) ENGINE=InnoDB;
```

### 3.3 缓存策略设计
三级缓存架构：
1. **内存缓存**：Redis存储热点查询结果，TTL=5分钟
2. **数据库缓存**：statistics_cache表存储中等热度数据，TTL=1小时
3. **预计算缓存**：预聚合表存储历史数据，每日更新

**Redis缓存键设计**：
- `report:query:{md5_hash}` - 报表查询结果
- `stats:device:{station_id}:{date}` - 设备统计
- `dashboard:{dashboard_id}:data` - 仪表盘数据

## 4. 数据一致性保障

### 4.1 外键约束
```sql
-- 报表模板与计划任务的关系
ALTER TABLE report_schedules 
ADD CONSTRAINT fk_report_schedules_template
FOREIGN KEY (template_id) REFERENCES report_templates(id)
ON DELETE CASCADE ON UPDATE CASCADE;

-- 执行记录与模板的关系
ALTER TABLE report_executions 
ADD CONSTRAINT fk_report_executions_template
FOREIGN KEY (template_id) REFERENCES report_templates(id)
ON DELETE RESTRICT ON UPDATE CASCADE;

-- 收藏与用户的关系
ALTER TABLE report_favorites 
ADD CONSTRAINT fk_report_favorites_user
FOREIGN KEY (user_id) REFERENCES users(id)
ON DELETE CASCADE ON UPDATE CASCADE;
```

### 4.2 数据清理策略
1. **执行记录保留**：成功记录保留90天，失败记录保留30天
2. **缓存自动清理**：Redis定期清理过期键，数据库缓存按TTL清理
3. **临时文件清理**：每日清理24小时前的临时导出文件
