## 1. 设计概述

### 1.1 设计目标
巡检管理模块是NESOM系统的预防性维护核心，负责：
- 巡检计划制定和任务分配
- 巡检路线和检查项管理
- 现场巡检执行和数据采集
- 巡检结果分析和问题跟踪
- 巡检报告生成和绩效评估

### 1.2 设计原则
1. **移动端优先**：支持离线操作、现场拍照、GPS定位
2. **实时性**：支持实时任务同步和状态更新
3. **扩展性**：支持多种巡检类型和自定义检查项
4. **完整性**：外键约束保障数据一致性
5. **可追溯性**：完整记录巡检过程和数据

### 1.3 与其他模块关系
- **依赖模块**：用户权限管理（巡检人员、管理人员）、设备监控模块（检查点关联设备）
- **关联模块**：工单管理（巡检发现问题自动创建工单）
- **数据提供**：报表统计（巡检完成率、问题分布）

## 2. 表结构详细设计

### 2.1 巡检计划表 (inspection_plans)
巡检计划主表，定义周期性或临时的巡检计划。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID() | 主键 |
| plan_code | VARCHAR(50) | UNIQUE NOT NULL | - | 计划编码，唯一标识 |
| plan_name | VARCHAR(100) | NOT NULL | - | 计划名称 |
| description | TEXT | - | NULL | 计划描述 |
| inspection_type | ENUM('routine','special','emergency') | NOT NULL | 'routine' | 巡检类型：例行、专项、应急 |
| priority | ENUM('low','medium','high','critical') | NOT NULL | 'medium' | 优先级 |
| route_id | VARCHAR(36) | FOREIGN KEY | NULL | 巡检路线ID |
| template_id | VARCHAR(36) | FOREIGN KEY | NULL | 巡检模板ID |
| frequency_type | ENUM('daily','weekly','monthly','quarterly','yearly','custom') | NOT NULL | 'weekly' | 频率类型 |
| frequency_value | INT | - | 1 | 频率值（如每2周） |
| frequency_days | JSON | - | NULL | 具体执行日（周几、每月几号等） |
| start_date | DATE | NOT NULL | - | 计划开始日期 |
| end_date | DATE | - | NULL | 计划结束日期（NULL表示无限期） |
| start_time | TIME | - | NULL | 每天开始时间 |
| end_time | TIME | - | NULL | 每天结束时间 |
| estimated_duration | INT | - | NULL | 预计耗时（分钟） |
| auto_assign | BOOLEAN | NOT NULL | TRUE | 是否自动分配任务 |
| assign_strategy | ENUM('round_robin','skill_based','location_based','manual') | NOT NULL | 'round_robin' | 分配策略 |
| require_photo | BOOLEAN | NOT NULL | TRUE | 是否要求拍照 |
| require_gps | BOOLEAN | NOT NULL | TRUE | 是否要求GPS定位 |
| require_signature | BOOLEAN | NOT NULL | FALSE | 是否要求签名 |
| status | ENUM('draft','active','paused','completed','cancelled') | NOT NULL | 'draft' | 状态 |
| created_by | VARCHAR(36) | NOT NULL | - | 创建人ID |
| created_by_name | VARCHAR(100) | NOT NULL | - | 创建人姓名 |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**外键约束**：
- `FOREIGN KEY (route_id) REFERENCES inspection_routes(id) ON DELETE SET NULL`
- `FOREIGN KEY (template_id) REFERENCES inspection_templates(id) ON DELETE SET NULL`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 唯一索引：`UNIQUE KEY uk_plan_code (plan_code)`
- 查询索引：`INDEX idx_status (status)`, `INDEX idx_inspection_type (inspection_type)`, `INDEX idx_created_by (created_by)`
- 时间索引：`INDEX idx_start_date (start_date)`

**业务规则**：
1. plan_code格式：`INSP-PLAN-{年月}-{序列号}`
2. 状态流转：draft → active → (paused/completed/cancelled)
3. frequency_days为JSON数组，如`[1,3,5]`表示周一、三、五执行
4. 自动分配任务时，按assign_strategy分配给人

### 2.2 巡检路线表 (inspection_routes)
巡检路线定义，包含检查点顺序和路径信息。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID() | 主键 |
| route_code | VARCHAR(50) | UNIQUE NOT NULL | - | 路线编码 |
| route_name | VARCHAR(100) | NOT NULL | - | 路线名称 |
| description | TEXT | - | NULL | 路线描述 |
| route_type | ENUM('fixed','dynamic','custom') | NOT NULL | 'fixed' | 路线类型：固定、动态、自定义 |
| total_checkpoints | INT | NOT NULL | 0 | 总检查点数 |
| estimated_distance | DECIMAL(8,2) | - | NULL | 预估距离（公里） |
| estimated_duration | INT | NOT NULL | 0 | 预估耗时（分钟） |
| start_location | VARCHAR(255) | - | NULL | 起点位置描述 |
| end_location | VARCHAR(255) | - | NULL | 终点位置描述 |
| path_coordinates | JSON | - | NULL | 路径坐标点数组（用于地图展示） |
| optimize_algorithm | VARCHAR(50) | - | NULL | 路线优化算法 |
| station_id | VARCHAR(36) | FOREIGN KEY | NULL | 所属场站ID |
| area_id | VARCHAR(36) | - | NULL | 区域ID |
| is_active | BOOLEAN | NOT NULL | TRUE | 是否启用 |
| created_by | VARCHAR(36) | NOT NULL | - | 创建人ID |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**外键约束**：
- `FOREIGN KEY (station_id) REFERENCES stations(id) ON DELETE CASCADE`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 唯一索引：`UNIQUE KEY uk_route_code (route_code)`
- 查询索引：`INDEX idx_station (station_id)`, `INDEX idx_is_active (is_active)`

**业务规则**：
1. route_code格式：`ROUTE-{场站代码}-{序列号}`
2. path_coordinates格式：`[[lng1,lat1], [lng2,lat2], ...]`
3. 动态路线根据设备位置实时生成最优路径
4. 路线启用后，已分配的巡检计划仍使用原路线

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID() | 主键 |

| checkpoint_name | VARCHAR(100) | NOT NULL | - | 检查点名称 |

| route_id | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 所属路线ID |
| sequence | INT | NOT NULL | 0 | 在路线中的顺序 |
| checkpoint_type | ENUM('device','location','area','custom') | NOT NULL | 'device' | 检查点类型 |
| device_id | VARCHAR(36) | FOREIGN KEY | NULL | 关联设备ID（检查点类型为device时） |
| location_name | VARCHAR(255) | - | NULL | 位置名称 |
| longitude | DECIMAL(10,6) | - | NULL | 经度 |
| latitude | DECIMAL(10,6) | - | NULL | 纬度 |
| altitude | DECIMAL(8,2) | - | NULL | 海拔(m) |
| qr_code | VARCHAR(255) | - | NULL | 二维码内容 |
| nfc_tag | VARCHAR(100) | - | NULL | NFC标签ID |
| required_actions | JSON | - | NULL | 必做动作列表（拍照、测温等） |
| inspection_items | JSON | - | NULL | 检查项模板 |
| safety_instructions | TEXT | - | NULL | 安全注意事项 |
| estimated_duration | INT | NOT NULL | 5 | 预估检查耗时（分钟） |
| is_mandatory | BOOLEAN | NOT NULL | TRUE | 是否必检 |
| tolerance_radius | INT | - | 50 | 容差半径（米，GPS验证用） |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**外键约束**：
- `FOREIGN KEY (route_id) REFERENCES inspection_routes(id) ON DELETE CASCADE`
- `FOREIGN KEY (device_id) REFERENCES devices(id) ON DELETE SET NULL`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 唯一索引：`UNIQUE KEY uk_checkpoint_code (checkpoint_code)`
- 查询索引：`INDEX idx_route_sequence (route_id, sequence)`, `INDEX idx_device (device_id)`
- 空间索引：`SPATIAL INDEX idx_location (longitude, latitude)`（如支持）

**业务规则**：
1. checkpoint_code格式：`CP-{路线编码}-{序列号}`
2. 检查点顺序在路线内唯一，用于导航
3. required_actions为JSON数组，如`["photo", "temperature", "vibration"]`
4. inspection_items为JSON数组，定义检查项和标准
5. 设备检查点必须关联有效设备

### 2.4 巡检任务表 (inspection_tasks)
巡检任务实例，由巡检计划生成。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID() | 主键 |
| task_code | VARCHAR(50) | UNIQUE NOT NULL | - | 任务编码 |
| plan_id | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 所属计划ID |
| route_id | VARCHAR(36) | FOREIGN KEY NOT NULL | - | 巡检路线ID |
| assigned_to | VARCHAR(36) | FOREIGN KEY | NULL | 分配给（用户ID） |
| assigned_to_name | VARCHAR(100) | - | NULL | 分配人姓名 |
| assigned_at | DATETIME | - | NULL | 分配时间 |
| scheduled_date | DATE | NOT NULL INDEX | - | 计划执行日期 |
| scheduled_start_time | TIME | - | NULL | 计划开始时间 |
| scheduled_end_time | TIME | - | NULL | 计划结束时间 |
| actual_start_time | DATETIME | - | NULL | 实际开始时间 |
| actual_end_time | DATETIME | - | NULL | 实际结束时间 |
| duration_minutes | INT | - | NULL | 实际耗时（分钟） |
| status | ENUM('pending','assigned','in_progress','completed','cancelled','overdue') | NOT NULL INDEX | 'pending' | 任务状态 |
| completion_rate | DECIMAL(5,2) | - | 0.00 | 完成率（0-100） |
| total_checkpoints | INT | NOT NULL | 0 | 总检查点数 |
| completed_checkpoints | INT | NOT NULL | 0 | 已完成检查点数 |
| problem_count | INT | NOT NULL | 0 | 发现问题数 |
| priority | ENUM('low','medium','high','critical') | NOT NULL | 'medium' | 优先级（继承计划） |
| is_offline | BOOLEAN | NOT NULL | FALSE | 是否离线执行 |
| offline_sync_status | ENUM('pending','synced','conflict','error') | - | NULL | 离线同步状态 |
| last_sync_time | DATETIME | - | NULL | 最后同步时间 |
| gps_track | JSON | - | NULL | GPS轨迹数据 |
| notes | TEXT | - | NULL | 备注 |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**外键约束**：
- `FOREIGN KEY (plan_id) REFERENCES inspection_plans(id) ON DELETE CASCADE`
- `FOREIGN KEY (route_id) REFERENCES inspection_routes(id) ON DELETE RESTRICT`
- `FOREIGN KEY (assigned_to) REFERENCES users(id) ON DELETE SET NULL`

**索引设计**：
- 主键索引：`PRIMARY KEY (id)`
- 唯一索引：`UNIQUE KEY uk_task_code (task_code)`
- 查询索引：`INDEX idx_assigned_to_status (assigned_to, status)`, `INDEX idx_scheduled_date (scheduled_date)`, `INDEX idx_status (status)`
- 计划关联索引：`INDEX idx_plan_id (plan_id)`

**业务规则**：
1. task_code格式：`TASK-{年月日}-{序列号}`
2. 状态流转：pending → assigned → in_progress → completed
3. 超过scheduled_date未开始，自动标记为overdue
4. completion_rate = (completed_checkpoints / total_checkpoints) * 100
5. GPS轨迹格式：`[{"lng": x, "lat": y, "timestamp": "..."}, ...]`

4. 状态为warning/fault时，problem_description必填

1. schedule_code格式：`SCH-{用户工号}-{年月}`
2. schedule_data格式根据schedule_type不同：每日`["2026-04-01", "2026-04-02"]`，每周`[1,3,5]`（周一三五）
3. 排班用于智能任务分配算法
4. 人员技能用于匹配专业巡检任务

## 6. 索引和查询优化策略

### 6.1 高性能查询场景
1. **我的巡检任务**：`WHERE assigned_to=? AND status IN (?) AND scheduled_date=?` → 索引: (assigned_to, status, scheduled_date)
2. **巡检任务看板**：`WHERE status IN ('assigned','in_progress') AND scheduled_date BETWEEN ? AND ?` → 索引: (status, scheduled_date)
3. **巡检结果查询**：`WHERE task_id=? ORDER BY sequence` → 索引: (task_id, sequence)
4. **设备巡检历史**：`WHERE device_id=? AND overall_status IN ('warning','fault')` → 索引: (device_id, overall_status, completed_time)

### 6.2 分区策略
1. **巡检结果表**：按completed_time按月分区
2. **巡检任务表**：按scheduled_date按月分区
3. **历史数据归档**：12个月前数据移至历史表

### 6.3 查询优化建议
1. **避免全表扫描**：WHERE条件使用索引字段
2. **分页优化**：使用游标分页而非OFFSET
3. **聚合查询**：定期预计算巡检统计
4. **读写分离**：报表查询走只读副本

-- 巡检计划表
CREATE TABLE inspection_plans (
    id VARCHAR(36) PRIMARY KEY DEFAULT (UUID()),
    plan_code VARCHAR(50) UNIQUE NOT NULL,
    plan_name VARCHAR(100) NOT NULL,
    description TEXT,
    inspection_type ENUM('routine','special','emergency') NOT NULL DEFAULT 'routine',
    priority ENUM('low','medium','high','critical') NOT NULL DEFAULT 'medium',
    route_id VARCHAR(36),
    template_id VARCHAR(36),
    frequency_type ENUM('daily','weekly','monthly','quarterly','yearly','custom') NOT NULL DEFAULT 'weekly',
    frequency_value INT DEFAULT 1,
    frequency_days JSON,
    start_date DATE NOT NULL,
    end_date DATE,
    start_time TIME,
    end_time TIME,
    estimated_duration INT,
    auto_assign BOOLEAN NOT NULL DEFAULT TRUE,
    assign_strategy ENUM('round_robin','skill_based','location_based','manual') NOT NULL DEFAULT 'round_robin',
    require_photo BOOLEAN NOT NULL DEFAULT TRUE,
    require_gps BOOLEAN NOT NULL DEFAULT TRUE,
    require_signature BOOLEAN NOT NULL DEFAULT FALSE,
    status ENUM('draft','active','paused','completed','cancelled') NOT NULL DEFAULT 'draft',
    created_by VARCHAR(36) NOT NULL,
    created_by_name VARCHAR(100) NOT NULL,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_status (status),
    INDEX idx_inspection_type (inspection_type),
    INDEX idx_created_by (created_by),
    INDEX idx_start_date (start_date),
    FOREIGN KEY (route_id) REFERENCES inspection_routes(id) ON DELETE SET NULL,
    FOREIGN KEY (template_id) REFERENCES inspection_templates(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 巡检任务表
CREATE TABLE inspection_tasks (

    assigned_to_name VARCHAR(100),
    assigned_at DATETIME,
