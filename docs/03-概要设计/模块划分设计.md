## 1. 概述

本文档定义NESOM系统的8个核心MVP模块，明确各模块的职责、功能边界、数据模型和接口规范。模块设计遵循"高内聚、低耦合"原则，支持未来的独立部署和扩展。

### 1.1 模块设计原则
1. **单一职责**：每个模块只负责一个明确的业务领域
2. **接口明确**：模块间通过定义良好的API进行通信
3. **数据自治**：模块拥有自己的数据模型，避免直接数据库耦合
4. **可测试性**：模块可独立测试，不依赖外部服务

## 2. 模块详细设计

### 2.1 设备监控模块

#### 2.1.1 模块职责
- 设备信息管理（增删改查）
- 设备状态实时监控
- 数据采集与存储
- 异常检测与告警
- 设备历史数据查询
- 设备阈值配置

#### 2.1.2 核心功能
1. **设备注册**：支持多品牌设备接入
2. **状态监控**：实时显示设备运行状态
3. **数据采集**：定时采集设备运行数据
4. **阈值告警**：自定义告警规则，实时通知
5. **历史查询**：设备历史数据曲线展示
6. **设备类型管理**：设备分类树结构管理

#### 2.1.3 数据模型（实际实现）
- `device_types`：设备类型/分类树（UUID主键）
- `devices`：设备主数据（含站点、类型、参数等扩展字段，UUID主键）
- `device_metrics`：高频指标数据（BIGINT自增）
- `device_alerts`：告警（含状态机：active/acknowledged/resolved/closed）
- `device_thresholds`：阈值配置（设备级/类型级）
    metric_value DECIMAL(10, 2) NOT NULL,     -- 指标值
    collected_at DATETIME NOT NULL,           -- 采集时间
    INDEX idx_device_time (device_id, collected_at)
);

-- 设备告警表
CREATE TABLE device_alerts (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    device_id BIGINT NOT NULL,
    alert_type VARCHAR(50) NOT NULL,          -- 告警类型
    alert_level ENUM('info', 'warning', 'error', 'critical') NOT NULL,
    alert_message TEXT NOT NULL,              -- 告警信息
    alert_time DATETIME NOT NULL,             -- 告警时间
    resolved_time DATETIME,                   -- 解决时间
    status ENUM('active', 'resolved', 'acknowledged') DEFAULT 'active',
    FOREIGN KEY (device_id) REFERENCES devices(id)
);
```

#### 2.1.4 接口定义
- `GET /api/v1/devices` - 获取设备列表
- `GET /api/v1/devices/{id}` - 获取设备详情
- `POST /api/v1/devices` - 创建设备
- `PUT /api/v1/devices/{id}` - 更新设备
- `DELETE /api/v1/devices/{id}` - 删除设备
- `GET /api/v1/devices/{id}/metrics` - 获取设备指标数据
- `GET /api/v1/devices/{id}/alerts` - 获取设备告警
- `POST /api/v1/devices/{id}/alerts/{alert_id}/acknowledge` - 确认告警
- `POST /api/v1/devices/{id}/alerts/{alert_id}/resolve` - 解决告警

### 2.3 用户权限管理模块

#### 2.3.1 模块职责
- 用户账户管理
- 角色权限管理
- 认证与授权
- 操作日志审计
- Token刷新与会话管理

#### 2.3.2 核心功能
1. **用户管理**：用户注册、信息维护、状态管理
2. **角色管理**：定义角色和权限集合
3. **权限控制**：接口级别权限验证
4. **登录认证**：用户名密码、JWT Token
5. **审计日志**：记录用户关键操作
6. **Token刷新**：支持Refresh Token机制

#### 2.3.3 数据模型（实际实现）
- `users`：用户（UUID主键，含锁定、失败登录次数等安全字段）
- `roles`：角色（UUID主键，含角色类型、数据范围）
- `permissions`：权限（UUID主键，含module/resource/action）
- `user_roles`：用户-角色关联
- `role_permissions`：角色-权限关联
- `refresh_tokens`：刷新令牌（支持Token轮换和撤销）
- `audit_logs`：审计日志

#### 2.3.4 接口定义
- `POST /api/v1/auth/login` - 用户登录

- `POST /api/v1/auth/refresh` - 刷新Token
- `GET /api/v1/users` - 获取用户列表
- `GET /api/v1/users/{id}` - 获取用户详情
- `POST /api/v1/users` - 创建用户
- `PUT /api/v1/users/{id}` - 更新用户
- `DELETE /api/v1/users/{id}` - 删除用户
- `GET /api/v1/roles` - 获取角色列表
- `POST /api/v1/roles` - 创建角色
- `PUT /api/v1/roles/{id}` - 更新角色

## 3. 模块间依赖关系

### 3.1 依赖矩阵

| 模块 | 依赖模块 | 依赖类型 | 说明 |
|------|----------|----------|------|
| 设备监控 | 用户权限 | 强依赖 | 需要用户认证和权限验证 |
| 工单管理 | 设备监控、用户权限 | 强依赖 | 依赖设备信息和用户信息 |
| 巡检管理 | 设备监控、用户权限 | 强依赖 | 依赖设备列表和用户分配 |
| 备件管理 | 用户权限 | 强依赖 | 需要用户操作权限 |
| 报表统计 | 所有业务模块 | 弱依赖 | 读取各模块数据，可异步 |
| 系统配置 | 无 | 无 | 基础模块，无外部依赖 |

### 3.2 数据流图

```
用户请求 → API网关 → 认证授权 → 业务模块
                                    ↓
                            数据库操作 (MySQL)
                                    ↓
                            缓存操作 (Redis) - 可选
                                    ↓
                            文件存储 (MinIO) - 可选
                                    ↓
                            响应返回
```

### 3.3 接口调用关系
1. **前端调用后端**：统一的RESTful API接口
2. **模块间调用**：通过服务接口调用，避免直接数据库访问
3. **异步通信**：通过消息队列处理耗时操作

## 4. 开发规范

### 4.1 代码组织
```
backend/
├── src/
│   ├── modules/
│   │   ├── device/          # 设备监控模块
│   │   ├── workorder/       # 工单管理模块
│   │   ├── auth/           # 用户权限模块
│   │   ├── sparepart/      # 备件管理模块
│   │   ├── inspection/     # 巡检管理模块
│   │   ├── report/         # 报表统计模块
│   │   └── system/         # 系统配置模块
│   ├── shared/             # 共享代码
│   └── main.py            # 应用入口
```

### 4.2 接口规范
- **URL格式**：`/api/v1/{模块}/{资源}/{操作}`
- **HTTP方法**：GET（查询）、POST（创建）、PUT（更新）、DELETE（删除）
- **响应格式**：统一JSON格式，包含code、message、data字段
- **错误处理**：HTTP状态码 + 错误码 + 错误信息

### 4.3 数据库规范
- **表名**：小写+下划线，复数形式
- **字段名**：小写+下划线
- **主键**：`id BIGINT PRIMARY KEY AUTO_INCREMENT`
- **时间字段**：`created_at`、`updated_at`、`deleted_at`（软删除）
- **索引**：常用查询字段添加索引
