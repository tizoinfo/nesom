## 1. 概述

本文档定义NESOM系统的7个核心MVP模块，明确各模块的职责、功能边界、数据模型和接口规范。模块设计遵循"高内聚、低耦合"原则，支持未来的独立部署和扩展。

### 1.1 模块设计原则
1. **单一职责**：每个模块只负责一个明确的业务领域
2. **接口明确**：模块间通过定义良好的API进行通信
3. **数据自治**：模块拥有自己的数据模型，避免直接数据库耦合
4. **可测试性**：模块可独立测试，不依赖外部服务

## 2. 模块详细设计

### 2.1 设备监控模块

#### 2.1.1 模块职责
- 设备信息管理（增删改查）
- 设备状态实时监控
- 数据采集与存储
- 异常检测与告警
- 设备历史数据查询

#### 2.1.2 核心功能
1. **设备注册**：支持多品牌设备接入
2. **状态监控**：实时显示设备运行状态
3. **数据采集**：定时采集设备运行数据
4. **阈值告警**：自定义告警规则，实时通知
5. **历史查询**：设备历史数据曲线展示

#### 2.1.3 数据模型
```sql
-- 设备表
CREATE TABLE devices (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    device_code VARCHAR(50) UNIQUE NOT NULL,  -- 设备编码
    device_name VARCHAR(100) NOT NULL,        -- 设备名称
    device_type VARCHAR(50) NOT NULL,         -- 设备类型
    brand VARCHAR(50),                        -- 品牌
    model VARCHAR(50),                        -- 型号
    location VARCHAR(200),                    -- 安装位置
    status ENUM('online', 'offline', 'fault', 'maintenance') DEFAULT 'offline',
    last_heartbeat DATETIME,                  -- 最后心跳时间
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 设备数据表
CREATE TABLE device_metrics (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    device_id BIGINT NOT NULL,
    metric_type VARCHAR(50) NOT NULL,         -- 指标类型：电压、电流、温度等
    metric_value DECIMAL(10, 2) NOT NULL,     -- 指标值
    collected_at DATETIME NOT NULL,           -- 采集时间
    INDEX idx_device_time (device_id, collected_at)
);

-- 设备告警表
CREATE TABLE device_alerts (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    device_id BIGINT NOT NULL,
    alert_type VARCHAR(50) NOT NULL,          -- 告警类型
    alert_level ENUM('info', 'warning', 'error', 'critical') NOT NULL,
    alert_message TEXT NOT NULL,              -- 告警信息
    alert_time DATETIME NOT NULL,             -- 告警时间
    resolved_time DATETIME,                   -- 解决时间
    status ENUM('active', 'resolved', 'acknowledged') DEFAULT 'active',
    FOREIGN KEY (device_id) REFERENCES devices(id)
);
```

- `PUT /api/v1/devices/{id}` - 更新设备
- `DELETE /api/v1/devices/{id}` - 删除设备
- `GET /api/v1/devices/{id}/metrics` - 获取设备指标数据

#### 2.3.3 数据模型
```sql
-- 用户表
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,     -- 用户名
    password_hash VARCHAR(255) NOT NULL,      -- 密码哈希
    real_name VARCHAR(50) NOT NULL,           -- 真实姓名
    email VARCHAR(100) UNIQUE,                -- 邮箱
    phone VARCHAR(20),                        -- 电话
    department VARCHAR(100),                  -- 部门
    position VARCHAR(100),                    -- 职位
    status ENUM('active', 'inactive', 'locked') DEFAULT 'active',
    last_login_at DATETIME,                   -- 最后登录时间
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 角色表
CREATE TABLE roles (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    role_code VARCHAR(50) UNIQUE NOT NULL,    -- 角色编码
    role_name VARCHAR(100) NOT NULL,          -- 角色名称
    description TEXT,                         -- 角色描述
    is_system BOOLEAN DEFAULT FALSE,          -- 是否为系统角色
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 权限表
CREATE TABLE permissions (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    perm_code VARCHAR(100) UNIQUE NOT NULL,   -- 权限编码
    perm_name VARCHAR(100) NOT NULL,          -- 权限名称
    resource_type VARCHAR(50) NOT NULL,       -- 资源类型：device, workorder等
    action VARCHAR(50) NOT NULL,              -- 操作：create, read, update, delete
    description TEXT,                         -- 权限描述
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 用户角色关联表
CREATE TABLE user_roles (
    user_id BIGINT NOT NULL,
    role_id BIGINT NOT NULL,
    PRIMARY KEY (user_id, role_id),
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (role_id) REFERENCES roles(id)
);

-- 角色权限关联表
CREATE TABLE role_permissions (
    role_id BIGINT NOT NULL,
    perm_id BIGINT NOT NULL,
    PRIMARY KEY (role_id, perm_id),
    FOREIGN KEY (role_id) REFERENCES roles(id),
    FOREIGN KEY (perm_id) REFERENCES permissions(id)
);
```

- `GET /api/v1/system/logs` - 查看系统日志

## 3. 模块间依赖关系

### 3.1 依赖矩阵

| 模块 | 依赖模块 | 依赖类型 | 说明 |
|------|----------|----------|------|
| 设备监控 | 用户权限 | 强依赖 | 需要用户认证和权限验证 |
| 工单管理 | 设备监控、用户权限 | 强依赖 | 依赖设备信息和用户信息 |
| 巡检管理 | 设备监控、用户权限 | 强依赖 | 依赖设备列表和用户分配 |
| 备件管理 | 用户权限 | 强依赖 | 需要用户操作权限 |
| 报表统计 | 所有业务模块 | 弱依赖 | 读取各模块数据，可异步 |
| 系统配置 | 无 | 无 | 基础模块，无外部依赖 |

### 3.2 数据流图

```
用户请求 → API网关 → 认证授权 → 业务模块
                                    ↓
                            数据库操作 (MySQL)
                                    ↓
                            缓存操作 (Redis) - 可选
                                    ↓
                            文件存储 (MinIO) - 可选
                                    ↓
                            响应返回
```

### 3.3 接口调用关系
1. **前端调用后端**：统一的RESTful API接口
2. **模块间调用**：通过服务接口调用，避免直接数据库访问
3. **异步通信**：通过消息队列处理耗时操作

### 4.2 接口规范
- **URL格式**：`/api/v1/{模块}/{资源}/{操作}`
- **HTTP方法**：GET（查询）、POST（创建）、PUT（更新）、DELETE（删除）
- **响应格式**：统一JSON格式，包含code、message、data字段
- **错误处理**：HTTP状态码 + 错误码 + 错误信息

### 4.3 数据库规范
- **表名**：小写+下划线，复数形式
- **字段名**：小写+下划线
- **主键**：`id BIGINT PRIMARY KEY AUTO_INCREMENT`
- **时间字段**：`created_at`、`updated_at`、`deleted_at`（软删除）
- **索引**：常用查询字段添加索引
