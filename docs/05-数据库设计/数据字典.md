## 1. 通用约定

### 1.1 主键与外键

- **主键**：以现有详细设计为准，存在两类并存：
  - `id VARCHAR(36)`（UUID）用于多数业务主表
  - `id BIGINT AUTO_INCREMENT` 用于部分高频明细/日志表
- **外键字段命名**：
  - 关联 UUID 主表：`{实体}_id` 通常为 `VARCHAR(36)`（例：`device_id`、`station_id`）
  - 关联自增明细表：`{实体}_id` 为 `BIGINT`

### 1.2 时间字段

- `created_at`：创建时间
- `updated_at`：更新时间
- 业务时间字段按语义命名：`alert_time`、`collected_at`、`generation_time` 等

建议统一：
- 高吞吐表使用毫秒精度 `DATETIME(3)`（如指标、告警开始时间、审计日志）

### 1.3 编码字段（建议唯一）

| 字段 | 含义 | 示例 |
|---|---|---|
| `device_code` | 设备编码（业务唯一） | `INV-001` |
| `work_order_no` | 工单编号（业务唯一） | `WO-场站-20260409-0001` |
| `plan_code` | 巡检计划编号（业务唯一） | `IP-0001` |
| `task_code` | 巡检任务编号（业务唯一） | `IT-20260409-0001` |
| `template_code` | 报表模板编码（业务唯一） | `RPT-DEVICE-UPTIME` |
| `config_key` | 配置键（系统唯一） | `security.jwt.secret_key` |
| `alert_code` | 告警编码（业务唯一） | `ALERT-设备-时间戳-序列` |

## 2. 枚举值定义（关键字段）

## 3. 核心字段释义（按表）

### 3.1 `devices`

| 字段 | 类型 | 含义 |
|---|---|---|
| `device_code` | VARCHAR(50) | 设备业务唯一编码 |
| `station_id` | VARCHAR(36) | 所属场站（公共主数据，待统一） |
| `device_type_id` | VARCHAR(36) | 设备类型（关联 `device_types`） |
| `status` | ENUM | 设备状态（见枚举） |
| `last_heartbeat` | DATETIME | 最近心跳/在线时间 |

### 3.2 `device_metrics`

| 字段 | 类型 | 含义 |
|---|---|---|
| `metric_type` | VARCHAR(50) | 指标类型（如 voltage/current/temp/power 等） |
| `metric_value` | DECIMAL(12,4) | 指标值 |
| `metric_unit` | VARCHAR(20) | 单位（V/A/kW/℃/% 等） |
| `collected_at` | DATETIME(3) | 采集时间（业务时间，毫秒精度） |
| `received_at` | DATETIME(3) | 接收时间（系统时间） |

### 3.3 `device_alerts`

| 字段 | 类型 | 含义 |
|---|---|---|
| `alert_type` | VARCHAR(50) | 告警类型（如 over_temp / comm_lost 等） |
| `alert_level` | ENUM | 告警等级（见枚举） |
| `status` | ENUM | 告警状态（见枚举） |
| `alert_code` | VARCHAR(50) | 告警编码（业务唯一，防重复） |
| `start_time` | DATETIME(3) | 告警开始时间 |
| `end_time` | DATETIME(3) | 告警结束时间（可空） |
| `acknowledged_at` | DATETIME | 确认时间（可空） |
| `resolved_at` | DATETIME | 解决时间（可空） |

### 3.4 `work_orders`

| 字段 | 类型 | 含义 |
|---|---|---|
| `work_order_no` | VARCHAR(50) | 工单业务唯一编号 |
| `work_order_type` | ENUM | 工单类型（见枚举） |
| `priority` | ENUM | 优先级（见枚举） |
| `emergency_level` | ENUM | 紧急程度（见枚举） |
| `status` | ENUM | 状态（见枚举） |
| `station_id` | VARCHAR(36) | 所属场站（公共主数据，待统一） |
| `device_id` | VARCHAR(36) | 关联设备（可空） |
| `reported_by` | VARCHAR(36) | 上报人 |
| `assigned_to` | VARCHAR(36) | 执行人（可空） |
| `scheduled_end` | DATETIME | 计划完成时间（超时判定） |

### 3.5 工单明细/历史表（替代 `work_order_logs`）

| 字段 | 类型 | 含义 |
|---|---|---|
| `work_order_details` | - | 处理步骤（可记录前后状态、照片、备件、测试结果等） |
| `work_order_status_history` | - | 状态流转审计（old/new/原因/时间/操作者） |

### 3.6 `users` / `roles` / `permissions`

| 表 | 字段 | 含义 |
|---|---|---|
| `users` | `password_hash` | 密码哈希（不可存明文） |
| `roles` | `role_code` | 角色编码（业务唯一） |
| `permissions` | `perm_code` | 权限编码（业务唯一） |
| `permissions` | `module`/`resource`/`action` | 权限三元组（便于权限检查与生成） |

### 3.7 `spare_parts`

| 字段 | 含义 |
|---|---|
| `available_stock` / `reserved_stock` | 可用/预留（并发需要策略） |
| `inventory_snapshots.version` | 乐观锁版本号（防并发冲突） |

### 3.8 `inventory_details`

| 字段 | 含义 |
|---|---|
| `transaction_no` | 事务编号（唯一） |
| `quantity` | 数量（正入库、负出库） |
| `work_order_id` | 工单关联（出库常见） |

### 3.9 `inspection_*`

| 表 | 字段 | 含义 |
|---|---|---|
| `inspection_plans` | `frequency_days` | JSON（周几/每月几号等） |
| `inspection_tasks` | `offline_sync_status` | 离线同步状态 |
| `inspection_results` | `check_items` | JSON（检查项结果） |

### 3.10 `report_*` / `statistics_cache` / `dashboard_configs`

| 表 | 字段 | 含义 |
|---|---|---|
| `report_templates` | `data_source_config` | 数据源配置（SQL/API/custom，JSON） |
| `report_executions` | `output_files` | 输出文件列表（JSON） |
| `statistics_cache` | `cache_key` | 缓存键（建议 MD5） |

### 3.11 `sys_config` / `sys_dict`（多租户）

| 表 | 字段 | 含义 |
|---|---|---|
| `sys_config` | `tenant_id` | 租户ID（隔离维度） |
| `sys_dict` | `dict_type` | 字典类型 |
| `sys_dict` | `parent_id` | 树结构父节点 |
